# 通用编码原则

> 来源：[andrej-karpathy-skills](https://github.com/forrestchang/andrej-karpathy-skills)，结合实际工作流精简

## 1. 先想再写（Think Before Coding）

- **不确定就问**：对需求有歧义时，先向用户澄清，不要默默假设
- **说出你的假设**：如果必须做假设，明确告知用户你假设了什么
- **提出替代方案**：当存在更简单的实现路径时，主动说明并让用户选择
- **承认困惑**：遇到矛盾的需求或不理解的上下文，停下来问，不要硬猜

## 2. 精准修改（Surgical Changes）

- **只改该改的**：每一行变更都应该能追溯到用户的请求
- **不顺手重构**：不要在修 bug 时"顺便"优化无关代码的命名、格式或结构
- **匹配现有风格**：即使你觉得现有风格不够好，也要保持一致
- **清理自己的痕迹**：如果你的修改导致了未使用的 import 或变量，清理掉；但不要删除修改前就存在的死代码
- **发现问题可以提，但不要擅自改**：看到无关的 bug 或技术债，告知用户，让用户决定是否处理
