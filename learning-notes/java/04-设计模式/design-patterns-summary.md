# 设计模式总结

> 从 private-notes 提取的技术学习笔记

## 设计模式概述

设计模式是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结。它描述了在软件设计过程中的一些不断重复发生的问题，以及该问题的解决方案。

## 设计模式分类

### 创建型模式（5种）

用于描述"怎样创建对象"，主要特点是"将对象的创建与使用分离"。

1. **单例模式（Singleton）**
   - 确保一个类只有一个实例
   - 提供全局访问点

2. **原型模式（Prototype）**
   - 通过复制现有实例来创建新实例

3. **工厂方法模式（Factory Method）**
   - 定义一个创建对象的接口，让子类决定实例化哪一个类

4. **抽象工厂模式（Abstract Factory）**
   - 提供一个创建一系列相关或相互依赖对象的接口

5. **建造者模式（Builder）**
   - 将一个复杂对象的构建与它的表示分离

### 结构型模式（7种）

用于描述如何将类或对象按某种布局组成更大的结构。

1. **代理模式（Proxy）**
2. **适配器模式（Adapter）**
3. **桥接模式（Bridge）**
4. **装饰模式（Decorator）**
5. **外观模式（Facade）**
6. **享元模式（Flyweight）**
7. **组合模式（Composite）**

### 行为型模式（11种）

用于描述类或对象之间怎样相互协作共同完成单个对象无法单独完成的任务。

1. **模板方法模式（Template Method）**
2. **策略模式（Strategy）**
3. **命令模式（Command）**
4. **职责链模式（Chain of Responsibility）**
5. **状态模式（State）**
6. **观察者模式（Observer）**
7. **中介者模式（Mediator）**
8. **迭代器模式（Iterator）**
9. **访问者模式（Visitor）**
10. **备忘录模式（Memento）**
11. **解释器模式（Interpreter）**

## 设计原则

1. **单一职责原则（SRP）**
2. **开闭原则（OCP）**
3. **里氏替换原则（LSP）**
4. **依赖倒置原则（DIP）**
5. **接口隔离原则（ISP）**
6. **迪米特法则（LoD）**

## 学习建议

1. 理解每个模式的适用场景
2. 掌握模式的实现方式
3. 了解模式的优缺点
4. 在实际项目中应用
