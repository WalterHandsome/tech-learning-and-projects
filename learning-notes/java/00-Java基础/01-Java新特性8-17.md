## Java 8 新特性

### 1. Lambda表达式

Lambda表达式是Java 8最重要的特性之一，它允许我们将函数作为方法的参数，或者将代码本身当作数据处理。

**语法格式：**
```java
(parameters) -> expression
或
(parameters) -> { statements; }
```

**示例：**
```java
// 旧写法
Collections.sort(list, new Comparator<String>() {
    @Override
    public int compare(String s1, String s2) {
        return s1.compareTo(s2);
    }
});

// Lambda表达式写法
Collections.sort(list, (s1, s2) -> s1.compareTo(s2));
```

### 2. 函数式接口

函数式接口是只包含一个抽象方法的接口。Java 8提供了`@FunctionalInterface`注解来标识函数式接口。

**常用的函数式接口：**
- `Function<T, R>`: 接受一个参数，返回一个结果
- `Consumer<T>`: 接受一个参数，无返回值
- `Supplier<T>`: 无参数，返回一个结果
- `Predicate<T>`: 接受一个参数，返回boolean值

**示例：**
```java
@FunctionalInterface
public interface MyFunction {
    int apply(int x, int y);
}

MyFunction add = (x, y) -> x + y;
MyFunction multiply = (x, y) -> x * y;
```

### 3. Stream API

Stream API是Java 8中处理集合的关键抽象概念，它可以让你以一种声明的方式处理数据集合。

**Stream的特点：**
- 不是数据结构，不会存储数据
- 不会改变源数据，会返回一个新的Stream
- 延迟执行，只有在终端操作时才会执行
- 可以并行处理

**常用操作：**
```java
List<String> list = Arrays.asList("a", "b", "c", "d");

// 过滤
list.stream().filter(s -> s.startsWith("a")).forEach(System.out::println);

// 映射
list.stream().map(String::toUpperCase).forEach(System.out::println);

// 排序
list.stream().sorted().forEach(System.out::println);

// 收集
List<String> result = list.stream()
    .filter(s -> s.length() > 1)
    .collect(Collectors.toList());
```

### 4. 接口默认方法和静态方法

Java 8允许在接口中定义默认方法和静态方法。

**默认方法：**
```java
public interface MyInterface {
    default void defaultMethod() {
        System.out.println("Default method");
    }
}
```

**静态方法：**
```java
public interface MyInterface {
    static void staticMethod() {
        System.out.println("Static method");
    }
}
```

### 5. 新的日期和时间API

Java 8引入了新的日期和时间API（java.time包），解决了旧API的线程安全问题。

**主要类：**
- `LocalDate`: 日期（年月日）
- `LocalTime`: 时间（时分秒）
- `LocalDateTime`: 日期时间
- `ZonedDateTime`: 带时区的日期时间
- `Duration`: 时间间隔
- `Period`: 日期间隔

**示例：**
```java
// 获取当前日期
LocalDate today = LocalDate.now();

// 创建指定日期
LocalDate date = LocalDate.of(2024, 1, 1);

// 日期格式化
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd");
String formatted = date.format(formatter);

// 日期解析
LocalDate parsed = LocalDate.parse("2024-01-01", formatter);
```

### 6. Optional类

Optional类是一个容器类，可以保存类型T的值，或者仅仅保存null。Optional提供很多有用的方法，这样我们就不用显式进行空值检测。

**示例：**
```java
Optional<String> optional = Optional.of("value");
optional.ifPresent(System.out::println);
String value = optional.orElse("default");
```

### 7. 方法引用

方法引用是Lambda表达式的简化写法。

**四种方法引用类型：**
1. 静态方法引用：`ClassName::staticMethod`
2. 实例方法引用：`instance::instanceMethod`
3. 类的实例方法引用：`ClassName::instanceMethod`
4. 构造器引用：`ClassName::new`

**示例：**
```java
List<String> list = Arrays.asList("a", "b", "c");
list.forEach(System.out::println); // 方法引用
list.forEach(s -> System.out.println(s)); // Lambda表达式
```

### 8. 并发增强

Java 8在并发方面做了很多改进：
- `CompletableFuture`: 异步编程
- `StampedLock`: 新的锁机制
- `LongAdder`: 高并发计数器


### 2. String新增方法

- `strip()`: 去除首尾空白字符

## Java 21 新特性（LTS）

> Java 21 于 2023 年 9 月发布，是继 Java 17 之后的下一个长期支持版本，也是目前企业级开发的推荐版本。

### 1. Virtual Threads（虚拟线程，正式）

虚拟线程是 Java 21 最重要的特性，由 Project Loom 引入。虚拟线程是轻量级线程，由 JVM 调度而非操作系统，可以轻松创建数百万个并发线程。

**核心优势：**
- 轻量级：每个虚拟线程仅占用几 KB 内存（平台线程约 1MB）
- 高并发：可创建数百万个虚拟线程
- 编程模型简单：使用同步代码风格实现异步性能

```java
// 创建虚拟线程
Thread.startVirtualThread(() -> {
    System.out.println("运行在虚拟线程中: " + Thread.currentThread());
});

// 使用 ExecutorService
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    // 轻松提交百万级任务
    IntStream.range(0, 1_000_000).forEach(i -> {
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1));
            return i;
        });
    });
}

// Spring Boot 3.2+ 启用虚拟线程
// application.yml:
// spring.threads.virtual.enabled: true
```

### 2. Record Patterns（记录模式，正式）

允许对 Record 类型进行解构，与 `instanceof` 和 `switch` 配合使用。

```java
record Point(int x, int y) {}

// 解构 Record
if (obj instanceof Point(int x, int y)) {
    System.out.println("坐标: " + x + ", " + y);
}

// 嵌套解构
record Line(Point start, Point end) {}

if (obj instanceof Line(Point(int x1, int y1), Point(int x2, int y2))) {
    System.out.println("线段: (" + x1 + "," + y1 + ") -> (" + x2 + "," + y2 + ")");
}
```

### 3. Pattern Matching for switch（正式）

switch 表达式的模式匹配正式发布，支持类型模式、守卫条件和 null 处理。

```java
// 类型模式 + 守卫条件
String describe(Object obj) {
    return switch (obj) {
        case Integer i when i > 0 -> "正整数: " + i;
        case Integer i            -> "非正整数: " + i;
        case String s when s.isEmpty() -> "空字符串";
        case String s             -> "字符串: " + s;
        case null                 -> "null 值";
        default                   -> "其他类型";
    };
}
```

### 4. Sequenced Collections（有序集合）

新增三个接口：`SequencedCollection`、`SequencedSet`、`SequencedMap`，统一了有序集合的访问方式。

```java
// 统一的首尾元素访问
SequencedCollection<String> list = new ArrayList<>(List.of("a", "b", "c"));
list.getFirst();    // "a"
list.getLast();     // "c"
list.addFirst("z"); // 在头部添加
list.reversed();    // 反转视图

// SequencedMap
SequencedMap<String, Integer> map = new LinkedHashMap<>();
map.put("a", 1);
map.put("b", 2);
map.firstEntry();   // a=1
map.lastEntry();    // b=2
map.pollLastEntry(); // 移除并返回最后一个
```

## Java 24 新特性（最新版本）

> Java 24 于 2025 年 3 月 18 日在 JavaOne 2025 大会上发布，包含 24 个 JEP，是自 2018 年以来特性最多的版本。
> 来源：[Oracle Releases Java 24](https://www.oracle.com/news/announcement/oracle-releases-java-24-2025-03-18/)

### 1. Stream Gatherers（正式，JEP 485）

自定义 Stream 中间操作，填补了 `Collector`（终端操作）的对应空白。

```java
// 内置 Gatherer：固定窗口
List<List<Integer>> windows = Stream.of(1, 2, 3, 4, 5, 6)
    .gather(Gatherers.windowFixed(3))
    .toList();
// 结果: [[1, 2, 3], [4, 5, 6]]

// 内置 Gatherer：滑动窗口
List<List<Integer>> sliding = Stream.of(1, 2, 3, 4, 5)
    .gather(Gatherers.windowSliding(3))
    .toList();
// 结果: [[1, 2, 3], [2, 3, 4], [3, 4, 5]]

// 自定义 Gatherer：去重（按属性）
Gatherer<String, ?, String> distinctByLength = Gatherer.ofSequential(
    HashSet::new,
    (state, element, downstream) -> {
        if (state.add(element.length())) {
            downstream.push(element);
        }
        return true;
    }
);
```

### 2. Primitive Types in Patterns, instanceof, and switch（预览，JEP 488）

模式匹配扩展到基本类型，无需手动装箱。

```java
// switch 支持基本类型
String classify(int value) {
    return switch (value) {
        case 0 -> "零";
        case int i when i > 0 && i <= 100 -> "小正数";
        case int i when i > 100 -> "大正数";
        default -> "负数";
    };
}

// instanceof 支持基本类型
if (obj instanceof int i) {
    System.out.println("整数值: " + i);
}
```

### 3. Simple Source Files and Instance Main Methods（正式，JEP 495）

简化 Java 入门程序，无需类声明和 `public static void main`。

```java
// 最简单的 Java 程序（无需 class 声明）
void main() {
    println("Hello, Java 24!");
}
```

### 4. 后量子密码学（JEP 496/497）

Java 24 引入了抗量子计算攻击的密码学算法：
- **ML-KEM**（Module-Lattice-Based Key-Encapsulation Mechanism）：密钥封装
- **ML-DSA**（Module-Lattice-Based Digital Signature Algorithm）：数字签名

### 5. 其他重要特性

- **Flexible Constructor Bodies（第三次预览，JEP 492）**：构造函数中 `super()` 前可初始化字段
- **Module Import Declarations（第二次预览，JEP 494）**：简化模块导入

## 总结

Java 8-24 的演进主要围绕以下几个方面：
1. **函数式编程**：Lambda 表达式、Stream API、函数式接口、Stream Gatherers
2. **类型推断与模式匹配**：var 关键字、instanceof 模式匹配、switch 模式匹配、Record Patterns
3. **模块化**：模块系统、Module Import Declarations
4. **语法简化**：记录类、文本块、switch 表达式、Simple Source Files
5. **并发革命**：Virtual Threads、Structured Concurrency、Scoped Values
6. **性能优化**：ZGC 分代模式、Compact Object Headers、AOT 类加载
7. **API 增强**：日期时间 API、HTTP Client、Sequenced Collections、Foreign Function & Memory API
8. **安全增强**：后量子密码学（ML-KEM、ML-DSA）

Java 21 是当前企业级开发的最佳选择，Virtual Threads 彻底改变了 Java 并发编程模型。
