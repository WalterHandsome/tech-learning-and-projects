## 1. 存储方案对比

| 方案 | 容量 | 生命周期 | 作用域 | 同步/异步 |
|------|------|---------|--------|----------|
| Cookie | ~4KB | 可设过期时间 | 同源+路径 | 同步 |
| localStorage | ~5-10MB | 永久 | 同源 | 同步 |
| sessionStorage | ~5-10MB | 标签页关闭 | 同源+标签页 | 同步 |
| IndexedDB | 无限制 | 永久 | 同源 | 异步 |
| Cache API | 无限制 | 手动管理 | 同源 | 异步 |

## 2. Cookie

```javascript
// 设置 Cookie
document.cookie = 'name=张三; max-age=3600; path=/; Secure; SameSite=Lax';

// 读取 Cookie
function getCookie(name) {
  const match = document.cookie.match(new RegExp(`(^| )${name}=([^;]*)`));
  return match ? decodeURIComponent(match[2]) : null;
}

// 删除 Cookie
document.cookie = 'name=; max-age=0; path=/';

// Cookie 属性
// HttpOnly  — JS 不可访问（防 XSS）
// Secure    — 仅 HTTPS 传输
// SameSite  — Strict/Lax/None（防 CSRF）
// Domain    — 作用域名
// Path      — 作用路径
// Max-Age   — 过期秒数
// Expires   — 过期日期
```

## 3. localStorage / sessionStorage

```javascript
// 封装（带过期时间）
const storage = {
  set(key, value, ttl) {
    const item = { value, expire: ttl ? Date.now() + ttl : null };
    localStorage.setItem(key, JSON.stringify(item));
  },
  get(key) {
    const raw = localStorage.getItem(key);
    if (!raw) return null;
    const item = JSON.parse(raw);
    if (item.expire && Date.now() > item.expire) {
      localStorage.removeItem(key);
      return null;
    }
    return item.value;
  },
  remove(key) { localStorage.removeItem(key); },
  clear() { localStorage.clear(); },
};

// 监听存储变化（跨标签页通信）
window.addEventListener('storage', (e) => {
  console.log(`${e.key}: ${e.oldValue} → ${e.newValue}`);
});
```

## 4. IndexedDB

```javascript
// 使用 idb 库简化操作
import { openDB } from 'idb';

const db = await openDB('myDB', 1, {
  upgrade(db) {
    const store = db.createObjectStore('users', { keyPath: 'id', autoIncrement: true });
    store.createIndex('email', 'email', { unique: true });
  },
});

// CRUD
await db.add('users', { name: '张三', email: 'test@example.com' });
const user = await db.get('users', 1);
const all = await db.getAll('users');
await db.put('users', { id: 1, name: '李四', email: 'test@example.com' });
await db.delete('users', 1);

// 按索引查询
const user = await db.getFromIndex('users', 'email', 'test@example.com');
```

## 5. 选型建议

- 认证信息（Token）→ HttpOnly Cookie（最安全）或 localStorage
- 用户偏好（主题、语言）→ localStorage
- 表单临时数据 → sessionStorage
- 大量结构化数据 → IndexedDB
- 离线资源缓存 → Cache API + Service Worker

## 6. 2026 年存储方案更新

### 6.1 Storage API 配额管理

```javascript
// 查询存储配额和使用量
const estimate = await navigator.storage.estimate();
console.log(`已用: ${(estimate.usage / 1024 / 1024).toFixed(2)} MB`);
console.log(`配额: ${(estimate.quota / 1024 / 1024).toFixed(2)} MB`);
// 典型配额：磁盘空间的 60%（Chrome）或 50%（Firefox）

// 请求持久化存储（不会被浏览器自动清理）
const persisted = await navigator.storage.persist();
if (persisted) {
  console.log('存储已持久化，不会被自动清理');
}
```

### 6.2 Origin Private File System（OPFS）

OPFS 提供高性能的文件系统访问，适合需要频繁读写大文件的场景（如 SQLite WASM）：

```javascript
// 获取 OPFS 根目录
const root = await navigator.storage.getDirectory();

// 创建/打开文件
const fileHandle = await root.getFileHandle('data.db', { create: true });

// 在 Web Worker 中使用同步访问（高性能）
// worker.js
const root = await navigator.storage.getDirectory();
const handle = await root.getFileHandle('data.db', { create: true });
const accessHandle = await handle.createSyncAccessHandle();

// 同步读写（仅 Worker 中可用）
const buffer = new ArrayBuffer(1024);
accessHandle.read(buffer, { at: 0 });
accessHandle.write(new Uint8Array([1, 2, 3]), { at: 0 });
accessHandle.flush();
accessHandle.close();
```

### 6.3 更新后的选型建议（2026）

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 认证 Token | HttpOnly Cookie | 防 XSS，自动携带 |
| 用户偏好 | localStorage | 简单、持久 |
| 表单临时数据 | sessionStorage | 标签页隔离 |
| 结构化数据 | IndexedDB | 异步、无容量限制 |
| 离线资源 | Cache API + SW | 请求级缓存 |
| 大文件/数据库 | OPFS | 高性能同步 I/O |
| AI 模型缓存 | Cache API + OPFS | 大体积、持久化 |
