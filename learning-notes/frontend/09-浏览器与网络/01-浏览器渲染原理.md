# 浏览器渲染原理

> Author: Walter Wang

## 1. 关键渲染路径

```
HTML → DOM 树
              ↘
               → 渲染树 → 布局(Layout) → 绘制(Paint) → 合成(Composite)
              ↗
CSS  → CSSOM
```

1. 解析 HTML → 构建 DOM 树
2. 解析 CSS → 构建 CSSOM 树
3. 合并 DOM + CSSOM → 渲染树（不包含 display:none 的元素）
4. 布局（Layout/Reflow）：计算每个节点的位置和大小
5. 绘制（Paint）：将节点绘制到图层
6. 合成（Composite）：将多个图层合成最终画面

## 2. 重排（Reflow）与重绘（Repaint）

```javascript
// 触发重排的操作（代价高）：
// - 修改元素尺寸（width, height, padding, margin, border）
// - 修改元素位置（top, left, position）
// - 修改文档结构（添加/删除 DOM 节点）
// - 读取布局信息（offsetTop, scrollTop, clientWidth, getComputedStyle）
// - 窗口大小变化

// 触发重绘的操作（代价较低）：
// - 修改颜色（color, background-color）
// - 修改可见性（visibility）
// - 修改阴影（box-shadow）

// 仅触发合成的操作（代价最低，推荐用于动画）：
// - transform
// - opacity
// - will-change
```

## 3. 优化策略

```javascript
// 1. 批量 DOM 操作
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}
list.appendChild(fragment); // 一次性插入

// 2. 避免强制同步布局
// ❌ 读写交替（触发多次重排）
for (const el of elements) {
  const height = el.offsetHeight; // 读
  el.style.height = height + 10 + 'px'; // 写
}

// ✅ 先读后写
const heights = elements.map(el => el.offsetHeight); // 批量读
elements.forEach((el, i) => {
  el.style.height = heights[i] + 10 + 'px'; // 批量写
});

// 3. 使用 requestAnimationFrame
function animate() {
  element.style.transform = `translateX(${x}px)`;
  requestAnimationFrame(animate);
}
requestAnimationFrame(animate);

// 4. CSS contain 属性（限制重排范围）
.widget {
  contain: layout style paint; /* 或 contain: content */
}

// 5. content-visibility（跳过屏幕外元素的渲染）
.card {
  content-visibility: auto;
  contain-intrinsic-size: 0 200px;
}
```

## 4. 资源加载

```html
<!-- CSS 阻塞渲染，应放在 head 中 -->
<link rel="stylesheet" href="style.css">

<!-- JS 阻塞 DOM 解析 -->
<script src="app.js"></script>

<!-- defer：DOM 解析完成后执行，保持顺序 -->
<script defer src="app.js"></script>

<!-- async：下载完立即执行，不保证顺序 -->
<script async src="analytics.js"></script>

<!-- 预加载 -->
<link rel="preload" href="font.woff2" as="font" crossorigin>
<link rel="prefetch" href="next-page.js">
<link rel="preconnect" href="https://api.example.com">
<link rel="dns-prefetch" href="https://cdn.example.com">
```

## 5. View Transitions API（2026 年跨浏览器支持）

Interop 2026 项目（Apple、Google、Microsoft、Mozilla 联合推动）在 2026 年 Q1 将**跨文档 View Transitions** 推送到所有主流浏览器。

来源：[Announcing Interop 2026](https://webkit.org/blog/17818/announcing-interop-2026/)

### 5.1 跨文档 View Transitions（MPA）

无需 JavaScript 框架即可实现 SPA 级别的页面过渡动画：

```css
/* 在两个页面的 CSS 中都添加 */
@view-transition {
  navigation: auto;  /* 同源导航自动启用过渡 */
}

/* 定义过渡元素 */
.hero-image {
  view-transition-name: hero;
}

/* 自定义过渡动画 */
::view-transition-old(hero) {
  animation: fade-out 0.3s ease-out;
}
::view-transition-new(hero) {
  animation: fade-in 0.3s ease-in;
}
```

来源：[Cross-Document View Transitions Guide](https://loke.dev/blog/cross-document-view-transitions-guide)

### 5.2 同文档 View Transitions（SPA）

```javascript
// SPA 中的状态切换动画
document.startViewTransition(() => {
  // 更新 DOM
  updateContent(newData);
});

// 带自定义动画的过渡
document.startViewTransition({
  update: () => updateContent(newData),
  types: ['slide-left'],  // 传递过渡类型
});
```

### 5.3 元素级 View Transitions（Chrome 147+）

```css
/* 元素级过渡：允许多个过渡同时运行 */
.card {
  view-transition-name: card;
  view-transition-class: card;
}

/* 元素级过渡不会阻塞页面其他部分的交互 */
```

来源：[Element-Scoped View Transitions](https://developer.chrome.com/docs/css-ui/view-transitions/element-scoped-view-transitions)

## 6. WebMCP：网站向 AI Agent 暴露工具

WebMCP（Web Model Context Protocol）是 2026 年 2 月 Chrome 146 引入的新标准，让网站以结构化方式向浏览器内 AI Agent 暴露可调用的工具，替代 Agent "看截图猜按钮"的方式。

来源：[Chrome WebMCP Early Preview](https://developer.chrome.com/blog/webmcp-epp)

```html
<!-- 声明式 API：将 HTML 表单暴露为 Agent 工具 -->
<form mcptool="search-products"
      mcpdescription="搜索商品目录">
  <input name="query" type="text" mcpdescription="搜索关键词">
  <input name="category" type="text" mcpdescription="商品分类">
  <button type="submit">搜索</button>
</form>
```

```javascript
// 命令式 API：JavaScript 注册工具
navigator.ai.tools.register({
  name: "get-order-status",
  description: "查询订单状态",
  parameters: {
    type: "object",
    properties: {
      orderId: { type: "string", description: "订单号" }
    },
    required: ["orderId"]
  },
  handler: async ({ orderId }) => {
    const order = await fetchOrder(orderId);
    return { status: order.status, eta: order.eta };
  }
});
```

**当前状态**：Chrome 146+ Early Preview，需通过 `chrome://flags` 启用。W3C 标准化进行中。
