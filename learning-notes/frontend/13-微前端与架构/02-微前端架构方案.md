## 1. 什么是微前端

- 将前端应用拆分为多个独立的子应用
- 每个子应用可以独立开发、部署、运行
- 主应用负责路由分发和子应用加载
- 适合大型团队、多技术栈共存的场景

## 2. qiankun（基于 single-spa）

```javascript
// 主应用
import { registerMicroApps, start } from 'qiankun';

registerMicroApps([
  {
    name: 'react-app',
    entry: '//localhost:3001',
    container: '#subapp-container',
    activeRule: '/react',
  },
  {
    name: 'vue-app',
    entry: '//localhost:3002',
    container: '#subapp-container',
    activeRule: '/vue',
  },
]);

start({ sandbox: { strictStyleIsolation: true } });
```

## 3. Module Federation（Webpack 5）

```javascript
// 远程应用 webpack.config.js
const { ModuleFederationPlugin } = require('webpack').container;

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'remote_app',
      filename: 'remoteEntry.js',
      exposes: {
        './Button': './src/components/Button',
        './Header': './src/components/Header',
      },
      shared: { react: { singleton: true }, 'react-dom': { singleton: true } },
    }),
  ],
};

// 主应用
new ModuleFederationPlugin({
  name: 'host_app',
  remotes: {
    remote_app: 'remote_app@http://localhost:3001/remoteEntry.js',
  },
  shared: { react: { singleton: true }, 'react-dom': { singleton: true } },
});

// 使用远程组件
const RemoteButton = React.lazy(() => import('remote_app/Button'));
```

## 5. Module Federation 2.0 稳定版

<!-- version-check: Module Federation 2.0 stable (@module-federation/enhanced), checked 2026-04-23 -->

> 🔄 更新于 2026-04-23

Module Federation 2.0 于 2026 年 4 月发布稳定版，由 ByteDance Web Infra 团队与原作者 Zack Jackson 合作开发，是一次重大架构升级。

### 核心改进

**运行时与构建工具解耦**

MF 2.0 将运行时完全从底层构建工具中分离，支持跨构建工具使用：

| 构建工具 | 支持状态 |
|---------|---------|
| webpack | ✅ 原生支持 |
| Rspack | ✅ 原生支持 |
| Vite | ✅ 通过插件 |
| Rollup / Rolldown | ✅ 通过插件 |
| Rsbuild | ✅ 原生支持 |
| Metro (React Native) | ✅ 通过插件 |

框架集成：Next.js、Modern.js、Rspress、Storybook
UI 库支持：React、Vue、React Native

**动态 TypeScript 类型提示**

消费远程模块时自动获得类型信息，无需维护共享类型包：

```javascript
// 安装 MF 2.0 增强插件
// npm install @module-federation/enhanced

// webpack.config.js（主应用）
const { ModuleFederationPlugin } = require('@module-federation/enhanced/webpack');

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'host_app',
      remotes: {
        remote_app: 'remote_app@http://localhost:3001/mf-manifest.json',
      },
      shared: { react: { singleton: true }, 'react-dom': { singleton: true } },
    }),
  ],
};

// 开发时自动获得远程模块的 TypeScript 类型（热重载）
import RemoteButton from 'remote_app/Button'; // ← 有完整类型提示
```

**Node.js 运行时支持**

远程模块可被 SSR 层、BFF 服务和 Node 微服务消费，实现前后端统一的模块分发。

**Side Effect Scanner**

CLI 工具，静态分析构建产物，在集成前识别全局变量污染、事件监听器注册和 CSS 选择器作用域影响。

### 从 MF 1.x 迁移

```javascript
// 1.x（webpack 内置）
const { ModuleFederationPlugin } = require('webpack').container;

// 2.0（增强插件）
const { ModuleFederationPlugin } = require('@module-federation/enhanced/webpack');
// 配置格式基本兼容，支持增量迁移
```

### 微前端方案选型（2026）

| 方案 | 适用场景 | 构建工具依赖 | 复杂度 |
|------|---------|-------------|--------|
| Module Federation 2.0 | 大规模多团队、运行时共享 | 多构建工具 | 高 |
| qiankun | 多技术栈共存、路由级拆分 | 无 | 中 |
| import maps | 轻量级、标准化 | 无 | 低 |
| Monorepo + 包共享 | 同技术栈、编译时共享 | pnpm/Turborepo | 中 |
