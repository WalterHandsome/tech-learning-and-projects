## 2. 浏览器原理

```
Q: 从输入 URL 到页面显示的过程？
A:
1. DNS 解析（域名 → IP）
2. TCP 三次握手（+ TLS 握手）
3. 发送 HTTP 请求
4. 服务器处理请求，返回响应
5. 浏览器解析 HTML → DOM 树
6. 解析 CSS → CSSOM 树
7. 合并 → 渲染树
8. 布局（Layout）→ 绘制（Paint）→ 合成（Composite）
9. 执行 JavaScript

Q: 跨域解决方案？
A:
- CORS（服务端设置 Access-Control-Allow-Origin）
- 代理服务器（开发环境 Vite/Webpack proxy，生产环境 Nginx）
- JSONP（仅 GET，已过时）
- WebSocket（不受同源策略限制）
```

## 3. 工程化

```
Q: Webpack 和 Vite 的区别？
A:
- Webpack：打包所有模块后启动，HMR 需要重新构建
- Vite：利用浏览器原生 ESM，按需编译，启动极快
- Vite 开发体验更好，Webpack 生态更成熟
- Vite 生产环境用 Rollup 打包

Q: Tree Shaking 原理？
A:
- 基于 ES Module 的静态分析
- 标记未使用的导出（unused exports）
- 在压缩阶段移除未使用的代码
- 要求使用 import/export，不能用 require
- package.json 中 sideEffects 标记副作用文件
```

// 两数之和
function twoSum(nums, target) {
  const map = new Map();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (map.has(complement)) return [map.get(complement), i];
    map.set(nums[i], i);
  }
}

// 有效括号
function isValid(s) {
  const stack = [];
  const map = { ')': '(', ']': '[', '}': '{' };
  for (const c of s) {
    if ('([{'.includes(c)) stack.push(c);
    else if (stack.pop() !== map[c]) return false;
  }
  return stack.length === 0;
}

// 反转链表
function reverseList(head) {
  let prev = null, curr = head;
  while (curr) {
    const next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }
  return prev;
}

// 最大子数组和（Kadane）
function maxSubArray(nums) {
  let maxSum = nums[0], currentSum = nums[0];
  for (let i = 1; i < nums.length; i++) {
    currentSum = Math.max(nums[i], currentSum + nums[i]);
    maxSum = Math.max(maxSum, currentSum);
  }
  return maxSum;
}

// LRU 缓存
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }
  get(key) {
    if (!this.cache.has(key)) return -1;
    const value = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, value);
    return value;
  }
  put(key, value) {
    if (this.cache.has(key)) this.cache.delete(key);
    else if (this.cache.size >= this.capacity) {
      this.cache.delete(this.cache.keys().next().value);
    }
    this.cache.set(key, value);
  }
}

### 工程化与工具链

```
Q: Vite 8 和 Webpack 的核心区别？
A: Vite 8 使用 Rolldown（Rust 编写）替代了 esbuild + Rollup：
   - 开发环境：ESM 原生模块，按需编译，毫秒级启动
   - 生产环境：Rolldown 统一打包，1.6x~7.7x 构建加速
   - Vite+ Alpha：统一 dev/test/lint/format/build 到 `vp` CLI
   Webpack 仍有最大生态，但新项目推荐 Vite。

Q: ESLint 10 的 Flat Config 是什么？
A: ESLint 10.0.0（2026-02）完全移除了 .eslintrc 配置格式，
   强制使用 eslint.config.js（Flat Config）：
   - 单一配置文件，不再有 extends/overrides 嵌套
   - 配置即代码，支持动态逻辑
   - 所有项目必须迁移
   替代方案：Biome（一体化 lint + format）

Q: TypeScript 7.0 Go 编译器意味着什么？
A: TypeScript 7.0 用 Go 重写编译器：
   - 预计构建速度提升 10x
   - 是 TypeScript 历史上最大的架构变更
   - 当前稳定版 5.9，7.0 预计 2026 年底
   - 对开发者 API 兼容，无需改代码

Q: Module Federation 2.0 解决了什么问题？
A: MF 2.0（2026-04 稳定版）：
   - 运行时与构建工具解耦（支持 Vite/Rspack/Rollup 等 7 种）
   - 动态 TypeScript 类型提示（消除远程模块类型丢失）
   - Node.js 运行时支持（前后端统一模块分发）
   - 是微前端架构的核心基础设施
```

### 浏览器与性能

```
Q: 2026 年 Core Web Vitals 有什么变化？
A: Google March 2026 Core Update 改为站点级聚合评估：
   - INP 仍是最常失败指标（43% 站点未通过 200ms）
   - scheduler.yield() 是优化 INP 的新 API
   - HTTP/3 已被 38.8% 网站采用
   - Speculation Rules API 实现近即时页面导航

Q: View Transitions API 是什么？
A: 浏览器原生的页面过渡动画 API：
   - 同文档过渡：SPA 内页面切换动画
   - 跨文档过渡：MPA 无需 JS 框架即可实现 SPA 级过渡
   - Interop 2026 推动全浏览器支持
   - CSS @view-transition 规则控制动画

Q: Trusted Types 是什么？
A: 浏览器原生的 DOM XSS 防护机制（2026-02 Baseline）：
   - 强制要求类型化值（不能直接赋值字符串到 innerHTML）
   - 从根本上防止 DOM XSS
   - 所有主流浏览器支持
   - 通过 CSP 头启用：Content-Security-Policy: require-trusted-types-for 'script'
```
