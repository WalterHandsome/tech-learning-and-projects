## 1. Core Web Vitals

```
LCP（Largest Contentful Paint）— 最大内容绘制
  好: < 2.5s | 需改进: 2.5-4s | 差: > 4s
  衡量加载性能，最大可见元素的渲染时间

INP（Interaction to Next Paint）— 交互到下一次绘制
  好: < 200ms | 需改进: 200-500ms | 差: > 500ms
  衡量交互响应性（替代 FID）

CLS（Cumulative Layout Shift）— 累积布局偏移
  好: < 0.1 | 需改进: 0.1-0.25 | 差: > 0.25
  衡量视觉稳定性
```

## 2. 其他指标

```
FCP（First Contentful Paint）— 首次内容绘制
TTFB（Time to First Byte）— 首字节时间
TTI（Time to Interactive）— 可交互时间
TBT（Total Blocking Time）— 总阻塞时间
```

## 3. 度量工具

```javascript
// Performance API

onLCP(({ value }) => console.log('LCP:', value));
onINP(({ value }) => console.log('INP:', value));
onCLS(({ value }) => console.log('CLS:', value));

// 上报到监控平台
function sendToAnalytics(metric) {
  navigator.sendBeacon('/api/metrics', JSON.stringify(metric));
}
onLCP(sendToAnalytics);
onINP(sendToAnalytics);
onCLS(sendToAnalytics);
```

## 5. Core Web Vitals 2026 更新

<!-- version-check: Core Web Vitals 2026, Google March 2026 Core Update, checked 2026-04-24 -->

> 🔄 更新于 2026-04-24

### 5.1 Google March 2026 Core Update

2026 年 3 月 27 日，Google 发布了重大核心算法更新，对 Core Web Vitals 的评估方式做出关键调整：

**从页面级评估变为站点级聚合**：CWV 评估不再仅看单个页面，而是聚合整个站点的表现。几个慢页面或高 CLS 的模板可能拖累整个域名的排名。

来源：[Core Web Vitals 2026 Guide](https://rishikc.com/articles/core-web-vitals-guide-2026-lcp-inp-cls/)、[March 2026 Update Impact](https://logoswebdesigns.com/blog/core-web-vitals-2026-march-update/)

**受影响站点数据**：早期数据显示受影响站点流量下降 20-35%，部分域名最差页面流量下降超过 50%。

### 5.2 INP 仍是最常失败的指标

INP（2024 年 3 月替代 FID）在 2026 年仍然是最具挑战性的指标：

- **43% 的站点未通过 200ms 阈值**
- INP 衡量整个页面访问期间所有交互的延迟，而非仅首次交互
- 复杂 SPA 应用（React/Vue）尤其容易超标

```javascript
// INP 优化：使用 scheduler.yield() 拆分长任务
async function handleClick() {
  // 第一步：立即更新 UI
  updateButtonState();

  // 让出主线程，避免阻塞下一次绘制
  await scheduler.yield();

  // 第二步：执行耗时计算
  const result = heavyComputation();

  await scheduler.yield();

  // 第三步：更新结果
  renderResult(result);
}

// 使用 web-vitals 库监控 INP 归因
import { onINP } from 'web-vitals/attribution';

onINP(({ value, attribution }) => {
  console.log('INP:', value, 'ms');
  console.log('事件类型:', attribution.eventType);
  console.log('目标元素:', attribution.eventTarget);
  console.log('输入延迟:', attribution.inputDelay);
  console.log('处理时间:', attribution.processingDuration);
  console.log('呈现延迟:', attribution.presentationDelay);
});
```

### 5.3 2026 年 CWV 阈值（未变）

| 指标 | 好 | 需改进 | 差 |
|------|-----|--------|-----|
| LCP | < 2.5s | 2.5-4s | > 4s |
| INP | < 200ms | 200-500ms | > 500ms |
| CLS | < 0.1 | 0.1-0.25 | > 0.25 |

> 注意：虽然阈值未变，但随着 HTTP/3 和 SSR 的普及，Google 对"好"的期望实际上在提高。2.4s 的 LCP 在 2023 年是安全的，在 2026 年已接近"需改进"边界。

### 5.4 2026 年优化建议

- **站点级思维**：不能只优化首页，所有模板页面都需要达标
- **INP 优先**：使用 `scheduler.yield()`、`requestIdleCallback`、Web Worker 拆分长任务
- **HTTP/3 + SSR**：利用 Next.js 16.2 / Nuxt 4 的 SSR 能力降低 LCP
- **图片优化**：AVIF 格式 + `loading="lazy"` + `fetchpriority="high"`（LCP 图片）
