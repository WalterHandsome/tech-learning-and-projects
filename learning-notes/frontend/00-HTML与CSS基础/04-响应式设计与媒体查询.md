## 2. 媒体查询

### 2.1 基本语法

```css
/* 最大宽度（移动优先推荐用 min-width） */
@media (max-width: 768px) {
  .sidebar { display: none; }
}

/* 最小宽度（移动优先策略） */
@media (min-width: 768px) {
  .container { max-width: 720px; }
}
@media (min-width: 1024px) {
  .container { max-width: 960px; }
}
@media (min-width: 1280px) {
  .container { max-width: 1200px; }
}

/* 组合条件 */
@media (min-width: 768px) and (max-width: 1023px) {
  /* 平板样式 */
}

/* 暗色模式 */
@media (prefers-color-scheme: dark) {
  body { background: #1a1a2e; color: #eee; }
}

/* 减少动画 */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}
```

### 2.2 常用断点

```css
/* 移动优先断点 */
/* 默认样式：手机（< 640px） */
@media (min-width: 640px)  { /* 大手机/小平板 */ }
@media (min-width: 768px)  { /* 平板 */ }
@media (min-width: 1024px) { /* 笔记本 */ }
@media (min-width: 1280px) { /* 桌面 */ }
@media (min-width: 1536px) { /* 大屏 */ }
```

## 3. 响应式单位

```css
.responsive {
  /* 相对单位 */
  font-size: 1rem;      /* 相对根元素字体大小 */
  padding: 1em;         /* 相对当前元素字体大小 */
  width: 50vw;          /* 视口宽度的50% */
  height: 100vh;        /* 视口高度的100% */
  height: 100dvh;       /* 动态视口高度（移动端推荐） */

  /* clamp() 响应式字体 */
  font-size: clamp(14px, 2.5vw, 20px);

  /* 容器查询单位 */
  font-size: 5cqi;      /* 容器内联尺寸的5% */
}
```

## 5. 容器查询（Container Queries）

```css
/* 定义容器 */
.card-container {
  container-type: inline-size;
  container-name: card;
}

/* 基于容器宽度的样式 */
@container card (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
}

@container card (max-width: 399px) {
  .card {
    display: flex;
    flex-direction: column;
  }
}
```


## 6. 容器查询深入（2026 更新）

<!-- version-check: Container Queries Baseline ~96%, Container Style Queries, checked 2026-05-02 -->

> 🔄 更新于 2026-05-02

容器查询（Container Queries）是 2026 年响应式设计的核心范式转变：从"基于视口"转向"基于容器"。全局浏览器覆盖率约 96%，已可在生产环境中放心使用。来源：[MDN Container Queries](https://developer.mozilla.org/docs/Web/CSS/CSS_Container_Queries)

### 6.1 容器查询 vs 媒体查询

```
┌─────────────────────────────────────────────────────┐
│  媒体查询（Media Queries）                           │
│  @media (min-width: 768px) { ... }                  │
│  → 基于视口（viewport）尺寸                          │
│  → 组件不知道自己在页面中的实际空间                    │
│                                                     │
│  容器查询（Container Queries）                       │
│  @container (min-width: 400px) { ... }              │
│  → 基于父容器尺寸                                    │
│  → 组件自适应可用空间，真正可移植                      │
└─────────────────────────────────────────────────────┘
```

### 6.2 完整用法

```css
/* 1. 声明容器 */
.sidebar {
  container-type: inline-size;  /* 监听内联方向（宽度）尺寸 */
  container-name: sidebar;
}

/* 简写 */
.main {
  container: main / inline-size;
}

/* 2. 基于容器尺寸的响应式样式 */
@container sidebar (min-width: 300px) {
  .nav-item {
    display: flex;
    gap: 8px;
  }
  .nav-label { display: inline; }
}

@container sidebar (max-width: 299px) {
  .nav-item {
    flex-direction: column;
    align-items: center;
  }
  .nav-label { display: none; } /* 窄侧边栏只显示图标 */
}

/* 3. 容器查询单位 */
.card-title {
  font-size: clamp(1rem, 5cqi, 1.5rem); /* cqi = 容器内联尺寸的 1% */
}
```

### 6.3 容器样式查询（Style Queries）

```css
/* 基于容器的自定义属性值来应用样式 */
.card-container {
  --theme: light;
}

@container style(--theme: dark) {
  .card {
    background: #1a1a2e;
    color: #eee;
  }
}
```

### 6.4 2026 年响应式设计最佳实践

| 场景 | 推荐方案 | 说明 |
|------|---------|------|
| 页面级布局 | 媒体查询 | 整体布局仍依赖视口 |
| 可复用组件 | 容器查询 | 组件自适应可用空间 |
| 字体大小 | `clamp()` + `cqi` | 流式排版 |
| 弹出层定位 | Anchor Positioning | 替代 JS 定位库 |
| 入场动画 | `@starting-style` | 替代 JS 动画库 |

> 2026 年的响应式设计从"Responsive Design"演进为"Intrinsic Design"——组件根据自身可用空间自适应，而非依赖全局断点。来源：[LogRocket - Container Queries 2026](https://blog.logrocket.com/container-queries-2026)
