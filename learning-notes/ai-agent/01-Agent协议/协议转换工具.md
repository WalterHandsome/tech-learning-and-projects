## 1. 为什么需要协议转换

Agent 需要调用各种工具，但接口五花八门。协议转换工具将异构接口统一为 Agent 可用的标准接口。

```
转换全景：
  网站/二进制 ──→ Open-CLI ────→ 标准 CLI ──→ Agent
  源代码      ──→ CLI-Anything ─→ 标准 CLI ──→ Agent
  REST API   ──→ Higress ──────→ MCP Server → Agent
```

## 2. Open-CLI — 万物皆可 CLI

将任何网站或本地二进制转换为标准化 CLI，让 Agent 通过命令行调用任何工具。

```bash
# 将网站转换为 CLI
open-cli wrap https://translate.google.com \
  --name "translate" --description "Google 翻译"

# Agent 调用
translate --from en --to zh --text "Hello World"

# 将复杂二进制简化为标准 CLI
open-cli wrap /usr/local/bin/ffmpeg \
  --name "video-convert" --simplify

video-convert --input video.mp4 --format mp3
```

```
适用：无 API 的网站、参数复杂的 CLI 工具、遗留系统
```

## 3. CLI-Anything — 源代码变 CLI

将任意源代码自动转换为 CLI 接口，通过 AST 分析提取函数签名并生成命令。

```python
# 原始源代码 data_processor.py
def clean_data(input_file: str, output_file: str, remove_nulls: bool = True):
    """清洗数据文件"""
    import pandas as pd
    df = pd.read_csv(input_file)
    if remove_nulls:
        df = df.dropna()
    df.to_csv(output_file, index=False)

def merge_files(files: list[str], output: str):
    """合并多个 CSV 文件"""
    pass
```

```bash
# 自动生成 CLI
cli-anything generate data_processor.py

# Agent 直接调用
data-processor clean-data --input-file data.csv --output-file clean.csv
data-processor merge-files --files a.csv b.csv --output merged.csv
```

## 4. Higress — API 网关直通 MCP

阿里巴巴开源的 AI 原生 API 网关，将现有 REST API 直接转换为 MCP Server，无需写代码。

```
架构：Agent ──MCP──→ Higress 网关 ──REST──→ 后端 API
      自带路由、认证、限流能力
```

```yaml
# Higress MCP 转换配置
apiVersion: networking.higress.io/v1
kind: McpServer
metadata:
  name: weather-api
spec:
  tools:
    - name: get_weather
      description: "获取城市天气"
      inputSchema:
        type: object
        properties:
          city: { type: string, description: "城市名称" }
        required: ["city"]
      requestTemplate:
        method: GET
        url: "https://api.weather.com/v1/forecast"
        params:
          q: "{{city}}"
          appid: "{{env.WEATHER_API_KEY}}"
      responseTemplate:
        body: "城市：{{response.name}} 温度：{{response.main.temp}}°C"
```

## 5. 对比表

| 特性 | Open-CLI | CLI-Anything | Higress |
|------|----------|-------------|---------|
| 提出方 | 社区 | 港大 HKUDS | 阿里巴巴 |
| 输入 | 网站/二进制 | 源代码 | REST API |
| 输出 | 标准 CLI | 标准 CLI | MCP Server |
| 语言 | TypeScript | Python | Go/YAML |
| 转换方式 | 包装+简化 | AST 分析 | 网关代理 |
| 需要改代码 | 否 | 否 | 否 |
| 部署方式 | 本地 | 本地 | K8s/Docker |

## 6. 选型建议

```
├─ 有网站但无 API        → Open-CLI（网站 → CLI）
├─ 有代码但无接口        → CLI-Anything（源码 → CLI）
├─ 有 REST API 想接 MCP  → Higress（API → MCP Server）
├─ 需要企业级网关能力     → Higress（自带路由/认证/限流）
└─ 快速原型验证          → Open-CLI（最快上手）
```

## 7. 组合使用

```bash
# 场景：让 Agent 调用公司遗留系统
# 1. 内部网站 → CLI
open-cli wrap https://internal.company.com/hr --name "hr-tool"
# 2. 数据脚本 → CLI
cli-anything generate scripts/pipeline.py
# 3. REST API → MCP
# 配置 Higress（见上文）
# 4. Agent 统一调用所有工具
```
