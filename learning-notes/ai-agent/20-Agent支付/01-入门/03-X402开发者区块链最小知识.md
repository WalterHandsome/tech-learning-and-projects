## 一、钱包：你的链上银行账户

### 1.1 钱包到底是什么？

钱包不是一个 App，本质上就是**一对密钥**：

```text
┌─────────────────────────────────────────────────────┐
│                    钱包 = 一对密钥                     │
│                                                     │
│  私钥（Private Key）                                 │
│  ├── 一串随机数（256 位，64 个十六进制字符）            │
│  ├── 例：0xac0974bec39a17e36ba4a6b4d238ff944bacb4...  │
│  ├── 作用：签名交易（相当于"密码"或"印章"）            │
│  └── 绝对不能泄露！谁有私钥谁就能花钱                  │
│                                                     │
│  公钥 → 地址（Address）                               │
│  ├── 由私钥通过数学运算推导出来（不可逆推）             │
│  ├── 例：0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913  │
│  ├── 作用：收款地址（相当于"银行卡号"）                │
│  └── 可以公开，别人知道了也花不了你的钱                 │
│                                                     │
│  类比：                                               │
│  私钥 = 银行卡密码（绝对保密）                         │
│  地址 = 银行卡号（可以告诉别人，用来收款）              │
└─────────────────────────────────────────────────────┘
```

### 1.2 生活类比

```text
传统银行：
  你去银行开户 → 银行给你一个卡号 + 你设一个密码
  别人转钱给你 → 用你的卡号
  你转钱给别人 → 输入密码确认

区块链钱包：
  你在本地生成一对密钥 → 公钥推导出地址（卡号）+ 私钥（密码）
  别人转钱给你 → 用你的地址
  你转钱给别人 → 用私钥签名确认

最大区别：
  银行账户 → 银行帮你保管密码，忘了可以重置
  区块链钱包 → 没有"银行"，私钥丢了就永远丢了，没人能帮你找回
```

### 1.3 x402 中钱包的角色

```text
买方（Agent）：需要一个钱包，里面有 USDC，用来付费调用 API
卖方（API 提供者）：需要一个钱包地址，用来收款

买方钱包的操作：
  1. 持有 USDC（余额）
  2. 用私钥签名支付授权（"我同意付 $0.001 给 0xABC..."）
  3. 签名后发给 Facilitator，Facilitator 代为执行链上转账

卖方钱包的操作：
  1. 提供一个收款地址（payTo）
  2. 等着收钱就行
```

## 二、USDC：链上的美元

### 2.1 什么是 USDC？

```text
USDC（USD Coin）是一种"稳定币"：

  1 USDC ≈ 1 美元（始终保持 1:1 锚定）

  发行方：Circle 公司（美国持牌金融机构）
  背书：每发行 1 USDC，Circle 在银行里存 1 美元作为储备
  审计：每月由会计师事务所审计储备金

  类比：
  USDC 就像"链上的美元代金券"
  你给 Circle 1 美元 → Circle 给你 1 USDC
  你还给 Circle 1 USDC → Circle 还你 1 美元
```

### 2.2 为什么 x402 用 USDC 而不用 ETH？

```text
ETH 的问题：价格波动大
  今天 1 ETH = $3200
  明天 1 ETH = $2800
  你的 API 定价 0.001 ETH，实际收入每天都在变

USDC 的优势：价格稳定
  今天 1 USDC = $1.00
  明天 1 USDC = $1.00
  你的 API 定价 0.001 USDC = $0.001，永远不变

所以 x402 选择 USDC 作为支付代币：
  · 定价简单（直接用美元思维）
  · 收入可预测
  · 不需要担心汇率波动
```

### 2.3 USDC 的精度

```text
这是新手最容易踩的坑之一！

USDC 在链上用整数表示，精度是 6 位小数：

  1 USDC = 1,000,000（一百万个最小单位）
  0.001 USDC = 1,000（一千个最小单位）
  0.000001 USDC = 1（一个最小单位）

代码中的表示：
  ✅ 正确：amount = 1000    // 表示 0.001 USDC
  ❌ 错误：amount = 0.001   // 链上不支持小数！

  转换公式：
  链上金额 = 美元金额 × 1,000,000
  美元金额 = 链上金额 ÷ 1,000,000

  例子：
  $0.001 → 0.001 × 1,000,000 = 1,000
  $1.00  → 1.00 × 1,000,000 = 1,000,000
  $0.50  → 0.50 × 1,000,000 = 500,000

为什么不用小数？
  因为计算机处理浮点数会有精度丢失（0.1 + 0.2 ≠ 0.3）
  金融系统绝对不能有精度问题，所以用整数表示
  这和传统支付系统用"分"而不是"元"是一个道理
```

## 三、Gas：链上的手续费

### 3.1 什么是 Gas？

```text
每一笔链上操作（转账、调用合约）都需要消耗计算资源。
Gas 就是你为这些计算资源支付的费用。

类比：
  寄快递 → 你要付运费
  链上转账 → 你要付 Gas 费

Gas 费用 ETH 支付（不是 USDC）：
  这就像在中国寄快递必须用人民币付运费，不能用美元
  链上操作必须用链的原生代币（ETH）付 Gas，不能用 USDC
```

### 3.2 Gas 费贵吗？

```text
取决于你用哪条链：

  以太坊主网（L1）：一笔转账 $1-50（很贵！）
  Base（L2）：一笔转账 $0.001-0.01（极便宜）
  Solana：一笔转账 $0.001（极便宜）

x402 推荐使用 Base 链，因为：
  · Gas 极低（L2 的优势）
  · Coinbase 生态（x402 就是 Coinbase 推出的）
  · USDC 流动性好

所以你在 Base 上做 x402，Gas 费基本可以忽略不计。
```

### 3.3 x402 的 Gas 谁来付？

```text
这是 x402 最巧妙的设计之一：

  传统链上转账：
    发送方构造交易 → 发送方付 Gas → 链上执行

  x402 支付（EIP-3009 模式）：
    买方只需要"签名"（不发交易，不付 Gas）
    Facilitator 拿着签名去链上提交交易 → Facilitator 付 Gas
    买方钱包里不需要 ETH！

  这就是为什么 x402 对买方（Agent）特别友好：
    Agent 钱包里只需要有 USDC，不需要 ETH
    签名是免费的（只是用私钥做一次数学运算）
    Gas 由 Facilitator 承担

  但注意：如果 Agent 需要做非 x402 的链上操作
  （比如把 USDC 从一个钱包转到另一个钱包），
  那还是需要 ETH 付 Gas 的。
```

## 四、签名：不是转账，是授权

### 4.1 签名 ≠ 转账

```text
这是很多新手最困惑的地方。

签名（Sign）：
  用私钥对一段数据做数学运算，产生一个"签名值"
  这个过程完全在本地完成，不和区块链交互
  不花 Gas，不花钱，不产生链上交易
  类比：在一份合同上盖章（盖章本身不花钱）

转账（Transfer）：
  构造一笔交易，广播到区块链网络，等待确认
  需要 Gas 费
  会改变链上状态（余额变化）
  类比：银行实际执行转账（会扣手续费）

x402 的巧妙之处：
  买方只做"签名"（免费）
  Facilitator 拿着签名去做"转账"（Facilitator 付 Gas）
  买方的体验：签个名就完事了
```

### 4.2 EIP-712：结构化签名

```text
EIP-712 是以太坊的一个标准，定义了"怎么对结构化数据签名"。

普通签名：对一串乱码签名（用户看不懂签了什么）
EIP-712 签名：对一个有结构的数据签名（用户能看懂签了什么）

x402 中的 EIP-712 签名内容：

  "我（0xMyWallet）授权转账 1000 USDC
   给 0xMerchant
   在 Base 链上
   有效期从 2026-04-10 10:00 到 2026-04-10 10:01
   一次性编号：0xabc123"

  这个签名包含了：
  · 谁付钱（from）
  · 付给谁（to）
  · 付多少（value）
  · 在哪条链上（chainId）
  · 什么时候有效（validAfter / validBefore）
  · 唯一编号（nonce，防止重复使用）

  Facilitator 拿到这个签名后：
  1. 验证签名是否有效（数学验证）
  2. 检查付款方余额是否足够
  3. 调用 USDC 合约的 transferWithAuthorization 方法
  4. 链上执行转账
```

### 4.3 EIP-3009：USDC 的"签名转账"

```text
EIP-3009 是 USDC 合约内置的一个功能：

  普通 ERC-20 转账（两步）：
    步骤 1：approve（授权某人可以花我的钱）→ 一笔交易，付 Gas
    步骤 2：transferFrom（那个人实际转走钱）→ 又一笔交易，又付 Gas

  EIP-3009 转账（一步）：
    步骤 1：付款方离线签名一个 TransferWithAuthorization
    步骤 2：任何人（Facilitator）拿着这个签名提交到链上 → 一笔交易完成

  优势：
  · 付款方不需要发交易（不需要 ETH 付 Gas）
  · 只需要一笔链上交易（而不是两笔）
  · 签名中包含了所有信息（谁付、付给谁、多少、有效期）

  这就是 x402 "买方零 Gas"的技术基础。
```

## 五、区块链网络：主网 vs 测试网

### 5.1 两个世界

```text
主网（Mainnet）：
  · 真实的区块链网络
  · 用真钱（真 USDC、真 ETH）
  · 交易不可逆（转错了拿不回来）
  · 用于生产环境

测试网（Testnet）：
  · 模拟的区块链网络，和主网功能完全一样
  · 用假钱（测试 USDC、测试 ETH）
  · 可以随便折腾，不会有任何损失
  · 用于开发和测试

  类比：
  主网 = 正式银行系统（真钱）
  测试网 = 银行的沙箱环境（模拟钱）

x402 开发中用到的网络：

  开发/测试：
  · Base Sepolia（Base 的测试网）
  · 网络标识：eip155:84532
  · 测试 USDC 地址：0x036CbD53842c5426634e7929541eC2318f3dCF7e

  · Base Mainnet（Base 主网）
  · 网络标识：eip155:8453
```

## 六、RPC 节点：你的代码怎么和区块链通信

### 6.1 什么是 RPC？

```text
RPC（Remote Procedure Call）= 远程过程调用

你的代码不能直接和区块链"对话"。
你需要通过一个 RPC 节点作为中间人。

  你的代码 → RPC 节点 → 区块链网络

  类比：
  你不能直接和银行的核心系统通信
  你通过银行的 API（网银/手机银行）来操作
  RPC 节点就是区块链的"API 网关"

常见操作：
  查余额：你的代码 → RPC → 区块链 → 返回余额
  发交易：你的代码 → RPC → 区块链 → 交易被打包确认
  查交易状态：你的代码 → RPC → 区块链 → 返回确认/失败
```

### 6.2 RPC 提供商

```text
你不需要自己运行区块链节点（那很复杂）。
直接用第三方 RPC 提供商的服务：

  Alchemy：https://www.alchemy.com（推荐，免费额度大）
  QuickNode：https://www.quicknode.com
  Infura：https://www.infura.io

  使用方式：
  1. 注册账号
  2. 创建一个项目，选择 Base Sepolia 网络
  3. 获得一个 RPC URL，类似：
     https://base-sepolia.g.alchemy.com/v2/你的API_KEY
  4. 在代码中配置这个 URL

  代码中使用：
  const provider = new ethers.JsonRpcProvider(
    "https://base-sepolia.g.alchemy.com/v2/你的API_KEY"
  );
```

## 七、Facilitator：x402 的"中间人"

### 7.1 为什么需要 Facilitator？

```text
x402 支付流程中有三个角色：

  买方（Agent）：签名支付授权
  卖方（API 服务）：提供付费内容
  Facilitator：验证签名 + 执行链上结算

  为什么不让买方自己转账？
  · 买方需要有 ETH 付 Gas（增加门槛）
  · 买方需要等交易确认（增加延迟）
  · 买方可能发了签名但不转账（卖方收不到钱）

  Facilitator 的作用：
  1. 验证买方的签名是否有效
  2. 检查买方余额是否足够
  3. 提交链上交易（Facilitator 付 Gas）
  4. 确认交易成功后通知卖方
  5. 卖方确认收到钱后返回数据

  类比：
  买方 = 买家（签支票）
  卖方 = 卖家（收支票）
  Facilitator = 银行（验证支票 + 执行转账）
```

### 7.2 用哪个 Facilitator？

```text
开发/测试阶段：
  x402.org 提供的免费测试网 Facilitator
  URL：https://x402.org/facilitator
  无需注册，无需 API Key
  只支持测试网（Base Sepolia）

生产阶段：
  Coinbase CDP Facilitator（推荐）
  免费 1000 笔/月，之后 $0.001/笔
  需要注册 Coinbase Developer Platform 获取 API Key
  支持 Base 主网、Polygon、Solana
```

## 八、完整流程串联：一笔 x402 支付到底发生了什么

```text
把前面所有概念串起来，看一笔完整的 x402 支付：

  Agent（买方）                API 服务（卖方）           Facilitator
     │                            │                        │
     │  1. GET /api/weather       │                        │
     │ ─────────────────────────→ │                        │
     │                            │                        │
     │  2. HTTP 402               │                        │
     │     "请支付 0.001 USDC     │                        │
     │      到 0xMerchant         │                        │
     │      在 Base 链上"          │                        │
     │ ←───────────────────────── │                        │
     │                            │                        │
     │  3. Agent 本地操作：                                  │
     │     a. 解析 402 响应中的支付要求                      │
     │     b. 构造 EIP-712 结构化数据                        │
     │        （from, to, value, validBefore, nonce）       │
     │     c. 用私钥签名（本地完成，不花钱）                  │
     │                            │                        │
     │  4. GET /api/weather       │                        │
     │     Header: X-PAYMENT=签名  │                        │
     │ ─────────────────────────→ │                        │
     │                            │                        │
     │                            │  5. 转发签名给           │
     │                            │     Facilitator         │
     │                            │ ─────────────────────→  │
     │                            │                        │
     │                            │  6. Facilitator：       │
     │                            │     a. 验证签名有效性    │
     │                            │     b. 检查买方 USDC 余额│
     │                            │     c. 提交链上交易      │
     │                            │        （Facilitator    │
     │                            │         付 Gas）        │
     │                            │     d. 等待交易确认      │
     │                            │                        │
     │                            │  7. 返回"结算成功"       │
     │                            │ ←─────────────────────  │
     │                            │                        │
     │  8. HTTP 200               │                        │
     │     { "weather": "晴天" }   │                        │
     │ ←───────────────────────── │                        │
     │                            │                        │

  整个过程耗时：约 200ms - 2s
  Agent 花费：0.001 USDC（约 $0.001）
  Agent 需要的 ETH：0（Facilitator 代付 Gas）
```

## 九、常见坑和注意事项

```text
坑 1：USDC 精度搞错
  ❌ amount = 0.001（链上不支持小数）
  ✅ amount = 1000（0.001 × 10^6）
  记住：USDC 是 6 位小数，所有金额乘以 1,000,000

坑 2：测试网和主网地址搞混
  测试网 USDC：0x036CbD53842c5426634e7929541eC2318f3dCF7e
  主网 USDC：  0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913
  搞混了 → 交易会失败或钱转到错误的合约

坑 3：网络 ID 差一位数字
  Base Sepolia（测试网）：eip155:84532
  Base Mainnet（主网）：  eip155:8453
  差一个 "2"！配错了整个支付流程都跑不通

坑 4：私钥泄露
  ❌ 把私钥写在代码里提交到 Git
  ❌ 把私钥放在 .env 文件里不加 .gitignore
  ✅ 开发用测试网私钥（丢了无所谓）
  ✅ 生产用 KMS/MPC 管理（私钥不出现在代码中）

坑 5：忘记 Gas
  x402 支付本身不需要 ETH（Facilitator 代付）
  但如果你要做内部资金调拨（钱包间转 USDC），需要少量 ETH
  Base L2 上 0.001 ETH 够用很久
```

## 十、术语速查表

```text
| 术语                | 一句话解释                                    |
|--------------------|--------------------------------------------|
| 钱包（Wallet）      | 一对密钥（私钥+地址），用来管理链上资产          |
| 私钥（Private Key） | 钱包的"密码"，用来签名交易，绝对不能泄露        |
| 地址（Address）      | 钱包的"卡号"，用来收款，可以公开               |

| ETH                | 以太坊原生代币，用来付 Gas 费                   |
| Gas                | 链上操作的手续费，用 ETH 支付                   |
| Base               | 以太坊 L2 网络，Gas 极低，x402 首选链           |
| L2（Layer 2）       | 建在以太坊之上的快速通道，更快更便宜             |
| 签名（Sign）        | 用私钥对数据做数学运算，证明"我同意"             |
| EIP-712            | 结构化数据签名标准，让签名内容可读               |
| EIP-3009           | USDC 的签名转账功能，买方不需要付 Gas            |
| RPC 节点            | 代码和区块链通信的中间人                        |
| Facilitator        | x402 的结算服务，验证签名+执行链上转账           |
| 主网（Mainnet）     | 真实的区块链网络，用真钱                        |
| 测试网（Testnet）   | 模拟网络，用假钱，免费练手                      |
| Faucet（水龙头）    | 免费发放测试代币的服务                          |
| 交易哈希（txHash）  | 链上交易的唯一 ID，用来查询交易状态              |
| 合约地址            | 智能合约在链上的地址（USDC 就是一个智能合约）     |
| CAIP-2             | 链的标准标识格式（如 eip155:8453 = Base 主网）  |
| nonce              | 一次性编号，防止签名被重复使用                   |
```
