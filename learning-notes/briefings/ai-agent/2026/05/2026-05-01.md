### 1. PyTorch Lightning 供应链攻击：恶意依赖被植入 AI 训练库

- **来源**：[Shai-Hulud Themed Malware Found in the PyTorch Lightning AI Training Library](https://semgrep.dev/blog/2026/malicious-dependency-in-pytorch-lightning-used-for-ai-training/)
- **标签**：`供应链安全` `PyTorch` `恶意软件`
- **评分**：时效 5 / 一手 5 / 相关 4 / 实用 5 = 总分 19
- **摘要**：Semgrep 安全团队发现 PyTorch Lightning（广泛使用的 AI 训练框架）中被植入了以《沙丘》中 Shai-Hulud 命名的恶意依赖包。该恶意代码通过依赖链注入，影响所有使用该库的 AI 训练管道。HN 上获得 374 分和 127 条讨论。
- **影响**：所有使用 PyTorch Lightning 的 AI 项目需要立即审查依赖链，这是继 PyPI 恶意包事件后又一次针对 AI 生态的供应链攻击
- **行动建议**：立即检查项目中 PyTorch Lightning 的版本和依赖树，使用 `pip audit` 或 Semgrep 扫描；考虑锁定依赖版本并启用哈希校验

## 📄 值得关注的论文

### Think it, Run it: 自愈式多 Agent AI 自动生成 ML 管道

- **作者/机构**：arXiv 2604.27096
- **摘要**：提出五 Agent 系统架构，自动从数据集和自然语言目标生成端到端 ML 管道，集成代码驱动的 RAG 和自愈执行机制
- **关键贡献**：统一的多 Agent 架构实现 ML 管道全自动化，包含 profiling、intent parsing、微服务推荐、DAG 构建和执行五个 Agent
- **链接**：[arXiv](https://arxiv.org/abs/2604.27096)

### Step-level Optimization for Efficient Computer-use Agents

- **作者/机构**：arXiv 2604.27151
- **摘要**：针对计算机操作 Agent 的效率优化，提出步骤级优化策略，避免在每个交互步骤都调用大型多模态模型
- **关键贡献**：打破"每步都用大模型"的范式，通过智能判断何时需要调用大模型来大幅降低成本和延迟
- **链接**：[arXiv](https://arxiv.org/abs/2604.27151)

### When Your LLM Reaches End-of-Life: 生产系统模型迁移框架

- **作者/机构**：arXiv 2604.27082
- **摘要**：提出基于贝叶斯统计的 LLM 迁移框架，通过校准自动评估指标与人工判断的关系，在有限人工评估数据下实现可靠的模型对比
- **关键贡献**：解决了生产环境中 LLM 模型替换的实际痛点，已在服务 530 万月活用户的商业问答系统上验证
- **链接**：[arXiv](https://arxiv.org/abs/2604.27082)

### End-to-end Autonomous Scientific Discovery on a Real Optical Platform

- **作者/机构**：arXiv 2604.27092
- **摘要**：首次在真实物理系统（光学平台）上实现端到端自主科学发现，LLM Agent 自主完成从假设生成到实验验证的全流程
- **关键贡献**：从"辅助预定义工作流"到"真正自主发现"的突破，在真实物理系统上产出了非平凡的实验结果
- **链接**：[arXiv](https://arxiv.org/abs/2604.27092)
