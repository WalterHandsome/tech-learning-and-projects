### 1. DeepSeek V4 正式发布：1.6T 开源 MoE，SWE-bench 80.6%，7x 成本优势挑战闭源前沿

- **来源**：[DeepSeek 官方](https://chat.deepseek.com) / [TechCrunch](https://techcrunch.com/2026/04/24/deepseek-previews-new-ai-model-that-closes-the-gap-with-frontier-models/) / [The Next Web](https://thenextweb.com/news/deepseek-v4-pro-flash-launch-open-source) / [buildfastwithai](https://www.buildfastwithai.com/blogs/deepseek-v4-pro-review-2026)
- **标签**：`DeepSeek` `V4` `开源模型` `Agentic Coding` `MoE`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：DeepSeek 于 4 月 24 日正式发布 V4 Preview 系列，结束了长达数月的等待。两个变体：V4-Pro（1.6T 总参数/49B 活跃，33T tokens 训练）和 V4-Flash（284B 总参数/13B 活跃，32T tokens 训练），均为 MoE 架构，原生 1M 上下文窗口，MIT 开源协议。核心架构创新：混合注意力机制结合 Compressed Sparse Attention（CSA）和 Heavily Compressed Attention（HCA），在 1M 上下文下 V4-Pro 仅使用 V3.2 的 27% 推理 FLOPs 和 10% KV 缓存。V4-Pro 在 SWE-bench Verified 达到 80.6%（距 Claude Opus 4.6 仅 0.2 分），Codeforces 评分 3,206（人类排名第 23），整体性能介于 GPT-5.2 和 GPT-5.4 之间。API 定价 $3.48/M 输出 tokens，是 Claude $25 的 1/7。支持华为昇腾芯片，HuggingFace 开放权重下载。
- **影响**：这是 DeepSeek 继 R1 之后的又一次"价格屠杀"——在 Coding 基准上逼近闭源前沿（80.6% vs Claude 80.8%），但价格仅为 1/7。V4 的发布时机极具戏剧性：GPT-5.5 发布仅一天后，DeepSeek 就用开源模型证明了闭源前沿的"护城河"正在快速缩小。对 Agent 开发者来说，V4-Pro 的 1M 原生上下文 + 极低推理成本组合，使其成为长时间运行 Agent 工作流的理想选择。华为昇腾支持使 V4 成为中国自主芯片生态的标杆。V3/V3.2 将于 7 月 24 日退役。
- **行动建议**：(1) 通过 DeepSeek API 或 HuggingFace 评估 V4-Pro 在实际 Agent 项目中的表现；(2) 对比 V4-Pro（$3.48/M）与 GPT-5.5（$30/M）和 Claude Opus 4.7（$25/M）的成本效益；(3) 关注 V4 正式版（非 Preview）的发布时间线。

### 2. GPT-5.5 API 正式开放：开发者终于可以构建 Agent 应用

- **来源**：[OpenAI 社区](https://community.openai.com/t/gpt-5-5-is-here-available-in-codex-and-chatgpt-today/1379630) / [OpenAI 官方](https://openai.com/index/introducing-gpt-5-5) / [Appwrite](https://appwrite.io/blog/post/gpt-5-5-launch)
- **标签**：`OpenAI` `GPT-5.5` `API` `Agent`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：OpenAI 于 4 月 24 日正式开放 GPT-5.5 API 访问（4/23 发布时仅限 ChatGPT 和 Codex）。GPT-5.5 可通过 Responses API 和 Chat Completions API 使用，支持 1M 上下文窗口。GPT-5.5 Pro 也通过 Responses API 提供。API 定价：标准版 $5/$30（输入/输出），Pro 版 $30/$180 per 1M tokens。关键能力：Terminal-Bench 2.0 得分 82.7%，SWE-Bench Pro 58.6%，OSWorld-Verified 78.7%。OpenAI 强调 GPT-5.5 在相同延迟下比 GPT-5.4 更高效——完成相同 Codex 任务消耗的 token 更少。推理努力级别支持 xhigh/high/medium/low/non-reasoning 五档。
- **影响**：API 开放意味着开发者现在可以将 GPT-5.5 的 Agent 能力集成到自己的应用中。结合 04-24 简报中报道的 GPT-5.5 发布，这完成了从"产品内测"到"开发者平台"的最后一步。82.7% 的 Terminal-Bench 2.0 得分使 GPT-5.5 在复杂命令行工作流中显著领先 Claude Opus 4.7（69.4%）。但 $30/M 的输出定价是 DeepSeek V4 的 8.6 倍——开发者需要在性能和成本之间做出选择。社区对价格翻倍（相比 GPT-5.4）有明显不满。
- **行动建议**：(1) 通过 API 评估 GPT-5.5 在你的 Agent 工作流中的表现；(2) 对比 GPT-5.5 标准版和 Pro 版的性能差异，评估是否值得 6x 溢价；(3) 使用推理努力级别控制成本——非关键任务用 medium/low，关键任务用 xhigh。

### 3. 前沿模型"三天三发"：GPT-5.5 → DeepSeek V4 → 开发者面临史上最复杂的模型选择

- **来源**：[implicator.ai](https://www.implicator.ai/openai-locked-the-door-deepseek-undercut-the-price-3/) / [akitaonrails](https://akitaonrails.com/en/2026/04/24/llm-benchmarks-parte-3-deepseek-kimi-mimo/) / [thedeepview](https://www.thedeepview.com/articles/deepseek-v4-resets-expectations-for-frontier-ai)
- **标签**：`行业分析` `模型竞争` `Agent选型`
- **评分**：时效 5 / 一手 4 / 相关 5 / 实用 5 = 总分 19
- **摘要**：4 月 22-24 日三天内，AI 模型市场经历了前所未有的密集发布：Qwen 3.6-27B（4/22，开源稠密模型超越 397B MoE）→ GPT-5.5（4/23，闭源 Agent 旗舰）→ DeepSeek V4（4/24，开源 1.6T MoE 逼近闭源前沿）。akitaonrails 的独立 Coding 基准测试（22 个模型、8 维度评分）显示当前 Tier A 模型包括 Claude Opus 4.6/4.7、GPT-5.4 xHigh、DeepSeek V4-Pro 和 Kimi K2.6。implicator.ai 分析指出 OpenAI 和 DeepSeek 采取了截然相反的分发策略：OpenAI 先锁定产品再开放 API，DeepSeek 直接开源权重。
- **影响**：Agent 开发者现在面临史上最复杂的模型选择矩阵。按成本效益排序：DeepSeek V4-Pro（$3.48/M，SWE-bench 80.6%）> Kimi K2.6（开源免费，SWE-bench 58.6%）> Qwen 3.6-27B（开源免费，本地可运行）> GPT-5.5（$30/M，Terminal-Bench 82.7%）> Claude Opus 4.7（$25/M，最强综合能力）。选择取决于场景：需要极致性能选 Claude/GPT-5.5，需要成本效益选 DeepSeek V4，需要本地部署选 Qwen 3.6-27B。
- **行动建议**：(1) 建立自己的 Agent 基准测试套件，在实际项目上对比这些模型；(2) 考虑混合策略——关键任务用 Claude/GPT-5.5，批量任务用 DeepSeek V4；(3) 关注 DeepSeek V4 正式版和 GPT-5.5 Pro API 的后续更新。

## 💬 社区讨论

### DeepSeek V4 vs GPT-5.5：开源与闭源的终极对决

DeepSeek V4 在 GPT-5.5 发布仅一天后登场，引发了社区关于开源 vs 闭源模型的激烈讨论。核心争议点：(1) V4-Pro 在 SWE-bench 上仅落后 Claude Opus 4.6 0.2 分，但价格是 1/7——"够用就好"的开发者会选择 DeepSeek；(2) GPT-5.5 在 Terminal-Bench 和计算机使用上仍有明显优势——需要极致 Agent 能力的场景仍需闭源模型；(3) DeepSeek 的华为昇腾支持使其成为中国开发者的首选——地缘政治因素正在影响技术选型；(4) swyx 的 AINews 分析认为 V4 整体水平约等于 Gemini 3.1/GPT-5.4/Opus 4.6，尚未达到 GPT-5.5 水平。

### GPT-5.5 定价争议：社区对价格翻倍表示不满

OpenAI 社区论坛上，开发者对 GPT-5.5 的定价策略表达了明显不满。从 GPT-5.1 到 GPT-5.4 再到 GPT-5.5，输出价格经历了连续翻倍。有开发者指出 service_tier:priority 的额外溢价使实际成本更高。OpenAI 回应称 GPT-5.5 完成相同任务消耗的 token 更少，净成本增幅约 20%，但社区对此说法持怀疑态度。
