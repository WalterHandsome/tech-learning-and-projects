### LangChain 为 Deep Agents 添加模型专属配置，性能提升 10-20 分

- **来源**：[Tuning Deep Agents to Work Well with Different Models](https://www.langchain.com/blog/tuning-deep-agents-different-models)
- **标签**：`LangGraph` `Deep Agents` `多模型适配`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 4 = 总分 19
- **摘要**：LangChain 宣布为 Deep Agents 引入模型专属 profile 机制，针对 OpenAI、Anthropic、Google 三大模型家族分别调整 prompt、工具调用和中间件配置。在 tau2-bench 子集上，相比通用配置提升了 10-20 分。
- **影响**：这意味着 Agent 框架正在从"一套 prompt 打天下"走向精细化的模型适配。对于在生产环境中使用 LangGraph 的开发者，切换模型不再是简单替换 API key，而是需要考虑模型特性的差异化调优。
- **行动建议**：如果你在用 LangGraph Deep Agents，升级到最新版并启用对应模型的 profile；如果自建 Agent，可以参考这个思路为不同模型维护独立的 prompt 模板。

### IBM 发布 Granite 4.1 系列：企业级开源 LLM 全面升级

- **来源**：[Introducing the IBM Granite 4.1 family of models](https://www.research.ibm.com/blog/granite-4-1-ai-foundation-models)
- **标签**：`IBM` `Granite` `开源模型` `Tool Calling`
- **评分**：时效 5 / 一手 5 / 相关 4 / 实用 4 = 总分 18
- **摘要**：IBM 发布 Granite 4.1 系列，包含 3B/8B/30B 三个规格的 dense decoder-only 模型，训练数据约 15T tokens，支持最长 512K 上下文。在 tool calling 和 instruction following 上显著超越 Granite 4.0，同时发布了 vision、speech、embedding 和 Guardian 安全模型。全部 Apache 2.0 开源。
- **影响**：对于需要在企业内网部署 Agent 的场景，Granite 4.1 的 tool calling 能力提升 + Apache 2.0 许可是一个值得关注的选项，尤其是 8B 和 30B 规格在成本和性能之间提供了不错的平衡。
- **行动建议**：如果你在评估企业私有化部署的 Agent 底座模型，值得在 Hugging Face 上拉取 Granite 4.1 跑一下 tool calling benchmark。

### Gartner 发布首份 Agentic AI Hype Cycle

- **来源**：[2026 Hype Cycle for Agentic AI](https://www.gartner.com/en/articles/hype-cycle-for-agentic-ai)
- **标签**：`Gartner` `行业报告` `Agentic AI`
- **评分**：时效 4 / 一手 5 / 相关 5 / 实用 3 = 总分 17
- **摘要**：Gartner 于 2026 年 4 月发布首份专门针对 Agentic AI 的 Hype Cycle，将 AI Agent 开发平台定位在"膨胀期望峰值"（Peak of Inflated Expectations），预计 2-5 年进入主流采用。同时预测 2026 年底 40% 的企业应用将集成任务专属 AI Agent（2025 年不到 5%）。
- **影响**：Gartner 单独为 Agentic AI 出 Hype Cycle 本身就说明这个领域的热度。但"膨胀期望峰值"的定位也意味着接下来可能进入幻灭低谷期，开发者需要关注哪些场景真正能落地。
- **行动建议**：在向管理层汇报 Agent 项目时可以引用这份报告；同时对"Agent 万能论"保持清醒，聚焦有明确 ROI 的场景。

## 📄 值得关注的论文

### Latent Agents: 将多智能体辩论蒸馏到单个 LLM

- **作者/机构**：arXiv
- **摘要**：提出一种两阶段微调方法，将多智能体辩论的推理能力蒸馏到单个 LLM 中，通过动态奖励调度和长度裁剪实现内化，大幅降低推理成本。
- **关键贡献**：解决了多智能体辩论虽然提升推理质量但计算开销巨大的问题，为生产环境中的 Agent 推理优化提供了新思路。
- **链接**：[arXiv:2604.24881](https://arxiv.org/abs/2604.24881)

### Leverage Laws: 人机协作的量化框架

- **作者/机构**：arXiv
- **摘要**：提出"杠杆比"（leverage ratio）概念来量化人机协作效率：Agent 替代的人类工作量 / 人类指定任务、处理中断和审查结果所需的时间。将信息需求分解为三个通道，各有独立的时间成本标量。
- **关键贡献**：为评估 Agent 实际价值提供了形式化框架，不再是模糊的"提效 X%"，而是可以按任务粒度计算 ROI。
- **链接**：[arXiv:2604.25040](https://arxiv.org/abs/2604.25040)

### Toward a Science of Intent: 开放世界 AI Agent 的委托边界

- **作者/机构**：arXiv
- **摘要**：提出"意图编译"（intent compilation）概念，将人类部分指定的目的转化为可检查的执行计划。定义了"闭合间隙"和"委托信封"两个核心概念，解释为什么能力强大的模型在开放机构中仍然难以部署。
- **关键贡献**：从理论层面解释了 Agent 部署的核心挑战不是能力而是信任和可检查性，对 Agent 产品设计有启发意义。
- **链接**：[arXiv:2604.25000](https://arxiv.org/abs/2604.25000)

- **来源**：[Ramp's Sheets AI Exfiltrates Financials](https://www.promptarmor.com/resources/ramps-sheets-ai-exfiltrates-financials)
- **标签**：`AI安全` `Prompt Injection` `数据泄露`
- **评分**：时效 5 / 一手 4 / 相关 4 / 实用 3 = 总分 16
- **摘要**：PromptArmor 披露 Ramp 的 Sheets AI 功能存在安全漏洞，可被利用泄露用户财务数据。这是又一个 AI 集成产品中 prompt injection 导致数据外泄的案例。
- **影响**：再次提醒：在 Agent 中集成外部数据源时，必须严格隔离用户数据和 LLM 可访问的上下文。
