### 1. OpenAI 模型、Codex 和 Managed Agents 正式登陆 AWS Bedrock

- **来源**：[OpenAI 官方](https://openai.com/index/openai-on-aws/) / [AWS 官方](https://aws.amazon.com/about-aws/whats-new/2026/04/bedrock-openai-models-codex-managed-agents/) / [Amazon 公告](https://www.aboutamazon.com/news/aws/bedrock-openai-models) / [CNBC](https://www.cnbc.com/2026/04/28/openai-brings-models-to-aws-after-ending-exclusivity-with-microsoft.html)
- **标签**：`OpenAI` `AWS` `Bedrock` `Codex` `Managed Agents` `多云`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：在 4 月 27 日与 Microsoft 结束独家授权仅一天后，OpenAI 即宣布 GPT 模型、Codex 编码 Agent 和 Managed Agents 正式登陆 Amazon Bedrock（限量预览）。GPT-5.4 已可用，GPT-5.5 即将上线。AWS CEO Matt Garman 在旧金山发布会上表示"这是客户长期以来的诉求"。Codex 目前拥有超过 400 万周活跃用户，登陆 AWS 将大幅扩展其企业覆盖面。Managed Agents 提供多步自动化、安全和治理能力。
- **影响**：这是 04-28 简报中 OpenAI-Microsoft 结束独家授权的直接落地。对 Agent 开发者意味着：(1) 在 AWS 上可以原生使用 OpenAI 模型构建 Agent，无需跨云调用；(2) Codex 作为编码 Agent 进入 AWS 生态，与 Amazon Q Developer 形成竞争/互补；(3) Managed Agents 在 Bedrock 上提供企业级 Agent 部署能力，降低了 Agent 生产化的门槛。AI 模型市场正式进入"多云时代"。
- **行动建议**：(1) 如果你的 Agent 部署在 AWS 上，立即申请 Bedrock OpenAI 模型的预览访问；(2) 评估 Codex on AWS 是否能替代你现有的编码辅助工具链；(3) 关注 Managed Agents 的 API 文档，评估与 LangGraph/CrewAI 的集成可能性。

### 2. NVIDIA 发布 Nemotron 3 Nano Omni：为 Agent 打造的开源多模态感知模型

- **来源**：[NVIDIA 官方博客](https://blogs.nvidia.com/blog/nemotron-3-nano-omni-multimodal-ai-agents/) / [NVIDIA 开发者博客](https://developer.nvidia.com/blog/nvidia-nemotron-3-nano-omni-powers-multimodal-agent-reasoning-in-a-single-efficient-open-model/) / [Hugging Face](https://huggingface.co/blog/nvidia/nemotron-3-nano-omni-multimodal-intelligence) / [The Next Web](https://thenextweb.com/news/nvidia-nemotron-nano-omni-multimodal-agent-edge)
- **标签**：`NVIDIA` `Nemotron` `多模态` `开源` `Agent感知` `MoE`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 4 = 总分 19
- **摘要**：NVIDIA 于 4 月 28 日发布 Nemotron 3 Nano Omni，一个开源多模态理解模型，采用 30B-A3B 混合专家（MoE）架构，仅 3B 参数激活即可处理视频、音频、图像和文本。该模型定位为 Agent 系统中的"多模态感知子 Agent"，替代传统的分散式视觉-语言-音频处理栈。关键指标：吞吐量比同类开源模型高 9 倍，在 6 个基准测试中领先。已在 Hugging Face、OpenRouter、NVIDIA Build 和 Amazon SageMaker JumpStart 上可用。
- **影响**：这是 Agent 多模态能力的重要基础设施升级：(1) 单一模型统一处理视觉、听觉和语言，大幅简化 Agent 的感知层架构；(2) 3B 激活参数意味着可以在边缘设备上运行，为本地 Agent 提供多模态能力；(3) 开源 + 多平台可用，降低了构建多模态 Agent 的门槛。与 Microsoft VibeVoice（语音专精）形成互补——NVIDIA 提供"全感知"，VibeVoice 提供"语音深度"。
- **行动建议**：(1) 如果你的 Agent 需要处理文档、音频或视频，评估 Nemotron 3 Nano Omni 作为感知层；(2) 在 Hugging Face 上试用，测试与你的 Agent 框架的集成；(3) 关注其在 SageMaker 上的部署方案，适合企业级 Agent 应用。

### 3. Anthropic 发布 Claude for Creative Work：9 个创意工具连接器

- **来源**：[Anthropic 官方](https://www.anthropic.com/news/claude-for-creative-work) / [Implicator](https://www.implicator.ai/anthropic-brings-claude-into-9-creative-tools-as-openai-and-gemini-push-images/) / [No Film School](https://nofilmschool.com/anthropic-claude-connectors-announced)
- **标签**：`Anthropic` `Claude` `创意工具` `Blender` `Adobe` `Ableton` `连接器`
- **评分**：时效 5 / 一手 5 / 相关 4 / 实用 4 = 总分 18
- **摘要**：Anthropic 发布 Claude for Creative Work，推出 9 个创意软件连接器，覆盖 Blender、Autodesk、Adobe、Ableton、Splice 等设计、音乐和 3D 工作领域。这些连接器让 Claude 能够直接在创意工具中编排多步工作流，而非仅作为文本助手。这是继 Claude Design（4 月 17 日发布的视觉设计产品）之后，Anthropic 在创意领域的又一重大布局。
- **影响**：Anthropic 正在将 Claude 从"对话 AI"扩展为"创意工作流 Agent"：(1) 连接器模式本质上是 MCP 在创意工具领域的落地——Claude 通过标准化接口操控外部工具；(2) 这为 Agent 开发者提供了一个参考范式：如何将 LLM Agent 嵌入专业领域工具链；(3) 与 Adobe 的 Firefly AI Assistant（4 月 15 日发布）形成竞合——Adobe 自建 Agent vs Anthropic 外接 Agent。
- **行动建议**：(1) 如果你在构建创意领域的 Agent，研究 Anthropic 连接器的架构设计；(2) 关注这些连接器是否基于 MCP 协议，可能为你的 Agent 提供复用机会。

## 📄 值得关注的论文

### PExA: 并行探索 Agent 用于复杂 Text-to-SQL

- **作者/机构**：arXiv 2604.22934
- **摘要**：将 Text-to-SQL 生成重新定义为软件测试覆盖问题。原始查询被分解为一组更简单的原子 SQL 测试用例，并行执行以确保语义覆盖。通过迭代测试用例覆盖来提升准确性，同时保持低延迟。
- **关键贡献**：提出了一种新颖的 Agent 架构范式——用"测试驱动"思维解决 NL2SQL 问题，并行探索显著降低了延迟-性能的权衡。对构建数据查询 Agent 有直接参考价值。
- **链接**：[arXiv](https://arxiv.org/abs/2604.22934)

### A Decoupled Human-in-the-Loop System for Controlled Autonomy in Agentic Workflows

- **作者/机构**：arXiv 2604.23049
- **摘要**：提出一种解耦的 HITL（Human-in-the-Loop）机制，将人类监督从应用逻辑中分离出来，提升复用性和一致性。现有 HITL 实现通常嵌入在应用代码中，限制了跨系统复用。
- **关键贡献**：与 OpenAI Symphony 的"持续式 Agent 编排"形成互补——Symphony 解决"何时启动 Agent"，这篇论文解决"Agent 运行时如何安全地引入人类决策"。对生产级 Agent 系统的安全控制有直接指导意义。
- **链接**：[arXiv](https://arxiv.org/abs/2604.23049)

### Don't Make the LLM Read the Graph: Make the Graph Think

- **作者/机构**：arXiv 2604.23057
- **摘要**：通过 3000+ 次 Hanabi 合作卡牌游戏实验（跨 4 个 LLM 家族），研究显式信念图是否能提升多 Agent 协作推理。发现集成架构决定了信念图的价值：作为 prompt 上下文时，图对强模型是"装饰品"，仅对弱模型在二阶心智理论任务上有帮助（80% vs 10%）。
- **关键贡献**：为多 Agent 系统中的知识表示提供了实证指导——不要盲目给 Agent 加图结构，要根据模型能力和任务复杂度选择。
- **链接**：[arXiv](https://arxiv.org/abs/2604.23057)

### Analytica: 基于软命题推理的 LLM Agent 分析架构

- **作者/机构**：arXiv 2604.23072
- **摘要**：提出 Analytica，一种基于"软命题推理"（SPR）的 Agent 架构。SPR 将复杂分析重构为结构化的命题估计过程，解决 LLM Agent 在金融预测、科学发现等复杂分析任务中推理不稳定、缺乏可验证组合结构的问题。
- **关键贡献**：为需要复杂推理的 Agent 提供了一种新的架构范式——将模糊的"分析"任务转化为可验证的命题链。
- **链接**：[arXiv](https://arxiv.org/abs/2604.23072)
