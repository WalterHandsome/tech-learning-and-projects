### 1. 小米 MiMo-V2.5 系列全面开源：MIT 协议、万亿参数、1M 上下文

- **来源**：[InfoQ](https://www.infoq.cn/article/YzBjivSoby8NR3gPWRtR) / [InfoWorld](https://www.infoworld.com/article/4164184/xiaomi-releases-mit%E2%80%91licensed-mimo-models-for-long%E2%80%91running-ai-agents.html) / [Hugging Face MiMo-V2.5-Pro](https://huggingface.co/XiaomiMiMo/MiMo-V2.5-Pro) / [The Standard](https://www.thestandard.com.hk/china/article/330592/Xiaomi-open-sources-MiMo-V25-Pro-model-under-MIT-license-topping-benchmarks)
- **标签**：`小米` `MiMo` `开源` `MIT` `MoE` `Agent`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：小米正式发布 MiMo-V2.5 系列大模型并全面开源，采用 MIT 协议。旗舰版 MiMo-V2.5-Pro 为 MoE 架构，总参数 1.02T（万亿），活跃参数 42B，支持 1M token 上下文窗口，采用混合注意力架构和 3 层多 Token 预测（MTP）。MiMo-V2.5 则是原生全模态模型，支持文本、图像、视频和音频理解。小米 AI 负责人罗福力主导此次发布，同时提供 100T 免费 token 供开发者试用。
- **影响**：小米在大模型领域的布局正在加速：(1) MIT 协议意味着完全开放商用，无需额外授权，这是国内大模型中最宽松的许可之一；(2) 万亿参数 + 1M 上下文 + Agent 能力优化，直接对标 DeepSeek V4 和 Qwen；(3) 但正如 InfoQ 标题所言"还是打不过 DeepSeek"——在编码和推理基准上仍有差距，定位更偏向"性价比开源选择"。对开发者而言，多了一个可商用的国产大模型选项。
- **行动建议**：(1) 在 Hugging Face 上试用 MiMo-V2.5-Pro，评估其在你的 Agent 场景中的表现；(2) 关注其 Agent 能力基准测试结果，与 DeepSeek V4 做对比。

### 2. 阿里 HappyHorse 1.0 开启灰测：AI 视频生成排名全球第一

- **来源**：[InfoQ](https://www.infoq.cn/article/3kfZZXewSZzErupjYGkT) / [fal.ai 官方](https://www.prnewswire.com/news-releases/fal-launches-happyhorse-1-0--the-1-ranked-ai-video-model-as-official-api-partner-302755003.html) / [Hugging Face](https://huggingface.co/happyhorseai/happy-horse-ai-video-generator) / [aibase](https://news.aibase.com/news/27526)
- **标签**：`阿里` `HappyHorse` `视频生成` `开源` `Elo排名`
- **评分**：时效 5 / 一手 4 / 相关 4 / 实用 4 = 总分 17
- **摘要**：阿里淘天未来生活实验室开发的 HappyHorse 1.0 视频生成模型正式开启灰测，720P 视频生成低至 0.44 元/秒。该模型为 150 亿参数开源模型，在 Artificial Analysis Video Arena 排行榜上同时拿下 Text-to-Video 和 Image-to-Video 双料第一（Elo 1333/1392），超越 Seedance 2.0、Veo 3.1 和 Sora 2。支持 1080P 超高清输出，可在单次推理中同时生成视频和同步音频。fal.ai 已作为官方 API 合作伙伴提供开发者和企业访问。
- **影响**：(1) 中国在 AI 视频生成领域取得了标志性突破——首次在国际盲测排行榜上全面超越 OpenAI Sora 和 Google Veo；(2) 开源 + 低价策略（0.44 元/秒）将大幅降低视频 AI 的使用门槛；(3) 由前快手 VP 张迪领衔，团队背景强大。
- **行动建议**：(1) 关注 HappyHorse API 的正式开放时间，评估在内容创作工作流中的应用；(2) 如果你在做视频相关的 Agent，这是一个值得集成的生成后端。

### 3. 网信办依法约谈剪映/即梦AI：AI 生成内容标识执法升级

- **来源**：[36氪氪星晚报](https://36kr.com/p/3786324189027332) / [PYMNTS](https://www.pymnts.com/cpi-posts/analysis-of-chinas-draft-administrative-measures-for-digital-virtual-human-information-services/)
- **标签**：`监管` `网信办` `AI标识` `剪映` `即梦AI` `合规`
- **评分**：时效 5 / 一手 4 / 相关 4 / 实用 3 = 总分 16
- **摘要**：国家互联网信息办公室近期发现"剪映""猫箱"App 及"即梦AI"网站存在未有效落实人工智能生成合成内容标识规定要求等问题，违反《网络安全法》《生成式人工智能服务管理暂行办法》《人工智能生成合成内容标识办法》等法律规定。网信办依法对上述平台采取约谈、责令改正、警告、从严处理责任人等处置处罚措施。
- **影响**：这是中国 AI 内容标识法规的首批重大执法行动：(1) 剪映（字节跳动旗下）和即梦AI 都是头部产品，被约谈释放了强监管信号；(2) 对 AI 应用开发者的直接影响——所有 AI 生成内容必须明确标识，这不是建议而是法律要求；(3) 结合 4 月发布的《数字虚拟人信息服务管理办法》草案，中国正在构建全球最细致的 AI 内容监管体系。
- **行动建议**：(1) 如果你的产品涉及 AI 生成内容，立即审查是否符合《人工智能生成合成内容标识办法》；(2) 关注后续是否有更多平台被约谈，了解执法标准。
