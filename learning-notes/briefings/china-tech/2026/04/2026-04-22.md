## 🔥 今日要闻（1-3 条）

### 1. Kimi K2.6 开源发布：1T 参数、300 子 Agent 并行、SWE-Bench Pro 58.6%

- **来源**：[SiliconAngle](https://siliconangle.com/2026/04/20/moonshot-ai-releases-kimi-k2-6-model-1t-parameters-attention-optimizations/) / [The Decoder](https://the-decoder.com/open-weight-kimi-k2-6-takes-on-gpt-5-4-and-claude-opus-4-6-with-agent-swarms/) / [roborhythms](https://www.roborhythms.com/kimi-k2-6-release/)
- **标签**：`月之暗面` `开源模型` `Agentic Coding`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：月之暗面（Moonshot AI）于 4 月 20 日开源发布 Kimi K2.6，这是一个 1T 参数的 MoE 模型（384 个专家，32B 活跃参数），256K 上下文窗口，原生多模态（文本/图像/视频）。核心亮点是 Agentic Coding 能力：可运行最多 300 个子 Agent 并行执行，在单次运行中完成 4,000+ 代码编辑，支持数小时甚至数天的长时间自主运行。SWE-Bench Pro 得分 58.6%，超越 GPT-5.4 和 Claude Opus 4.6。API 成本约为 Claude Opus 的 1/4。已在 Kimi.com、开发者 API、Kimi Code 和 Ollama 上可用。
- **影响**：K2.6 是中国开源模型在 Agentic Coding 领域的重大突破。300 子 Agent 并行 + 4,000 步协调执行的能力，意味着它不仅是一个代码生成模型，而是一个完整的软件工程 Agent 系统。开源 + 低成本的组合对 Claude Code 和 Codex 构成直接竞争威胁。对开发者来说，这是一个值得立即评估的生产级选择。
- **行动建议**：(1) 通过 Ollama 本地部署 K2.6 进行评估；(2) 在实际项目中对比 K2.6 与 Claude Code 的 Agentic Coding 表现；(3) 关注 API 定价，评估成本优势。

### 2. 阿里发布 Qwen 3.6-Max-Preview：闭源旗舰，Agentic Coding 全面超越 Plus

- **来源**：[Decrypt](https://decrypt.co/364948/alibaba-qwen-3-6-max-preview-most-powerful-model?amp=1) / [buildfastwithai](https://www.buildfastwithai.com/blogs/qwen3-6-max-preview-review-2026) / [therift.ai](https://www.therift.ai/news-feed/qwen-launches-qwen3-6-max-preview-with-enhanced-agentic-coding-and-knowledge)
- **标签**：`阿里巴巴` `Qwen` `闭源模型`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 4 = 总分 19
- **摘要**：阿里于 4 月 20 日发布 Qwen 3.6-Max-Preview，这是 Qwen 系列迄今最强的闭源旗舰模型。相比 3 月底发布的 Qwen 3.6-Plus，Max-Preview 在三个核心维度上显著提升：Agentic Coding（登顶 6 项编码基准）、世界知识和指令遵循。目前通过阿里云 Model Studio 和 Qwen Studio 提供预览访问。这标志着阿里也在走"开源 + 闭源"混合路线——3.6-Plus 免费开放，Max 作为付费旗舰。
- **影响**：结合 Meta Muse Spark 闭源转向（04-21），中国 AI 厂商也在复制"开源引流 + 闭源变现"的商业模式。Qwen 3.6-Max 的发布意味着阿里认为最强模型需要闭源来保护竞争优势和实现商业化。对开发者来说，3.6-Plus 仍然是免费的强力选择，但最强能力需要付费获取。
- **行动建议**：评估 Qwen 3.6-Max-Preview 在 Agentic Coding 场景的表现，与 3.6-Plus 做对比。关注正式版发布时间和定价。

### 3. 北京收紧 AI 出海管控：Manus 创始人被限制出境，Meta 收购案升级

- **来源**：[Washington Post](https://www.washingtonpost.com/world/2026/04/21/china-ai-competition-manus-meta/) / [The Decoder](https://the-decoder.com/beijing-brands-metas-manus-acquisition-as-conspiratorial-and-bars-founders-from-leaving-china/) / [letsdatascience](https://letsdatascience.com/news/beijing-blocks-ai-firms-from-shedding-chinese-ties-2e610681)
- **标签**：`政策监管` `AI出海` `Manus`
- **评分**：时效 5 / 一手 5 / 相关 4 / 实用 4 = 总分 18
- **摘要**：Washington Post 4 月 21 日报道，北京正在加强对试图"脱中"的 AI 创业公司的管控。最突出的案例是 Manus AI：这家 AI Agent 公司 2025 年夏天将总部从北京迁至新加坡，12 月被 Meta 以约 20 亿美元收购。中国发改委（NDRC）3 月召见了联合创始人肖弘和季逸超，随后限制两人出境。商务部等多部门正在审查该交易是否违反出口管制、用户数据保护和国家安全法规。监管机构正在调查 Manus 是否存在"洗模型"行为——将中国开发的 AI 系统重新包装或迁移到海外以规避国内管控。
- **影响**：这是中国 AI 产业政策的重大信号。北京正在明确划定红线：中国培育的 AI 技术和人才不能轻易流向海外竞争对手。对中国 AI 创业公司来说，"新加坡路线"（迁移总部到海外以获取国际资本和市场）的风险大幅上升。对国际投资者来说，收购中国 AI 公司的合规复杂度显著增加。
- **行动建议**：关注 Manus 案的最终裁决，它将成为中国 AI 出海政策的标杆案例。如果你的项目涉及中国 AI 模型或数据，评估出口管制合规风险。
