### 1. Linux "Copy Fail" 漏洞：9 年内核缺陷可让任意用户秒获 root 权限

- **来源**：[Copy Fail: Universal Linux Local Privilege Escalation Vulnerability](https://www.wiz.io/blog/copyfail-cve-2026-31431-linux-privilege-escalation-vulnerability)
- **标签**：`安全` `Linux` `CVE` `内核漏洞`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：CVE-2026-31431（"Copy Fail"）是 Linux 内核 algif_aead 模块中的本地提权漏洞，CVSS 评分 7.8。该漏洞自 2017 年引入，影响几乎所有主流 Linux 发行版。攻击者仅需一个 732 字节的 Python 脚本即可从普通用户提权到 root，无需特定发行版适配、无竞态条件、一次成功。由安全公司 Theori 使用 AI 辅助扫描发现。
- **影响**：所有运行 2017 年后 Linux 内核的服务器和容器环境都受影响，这是近年来最严重的 Linux 本地提权漏洞之一
- **行动建议**：立即检查所有 Linux 服务器和容器的内核版本；应用各发行版已发布的补丁；在补丁前可通过禁用 AF_ALG 模块缓解

### 3. Ubuntu 遭 DDoS 攻击，服务大面积中断

- **来源**：[Ubuntu services hit by outages after DDoS attack](https://techcrunch.com/2026/05/01/ubuntu-services-hit-by-outages-after-ddos-attack/)
- **标签**：`安全` `Ubuntu` `DDoS` `Linux`
- **评分**：时效 5 / 一手 4 / 相关 4 / 实用 4 = 总分 17
- **摘要**：黑客组织对 Ubuntu 和 Canonical 的多个网站发起 DDoS 攻击，导致服务中断，用户无法正常更新 Linux 系统。这是继 Copy Fail 漏洞之后，Linux 生态在同一周内遭遇的第二次重大安全事件。
- **影响**：依赖 Ubuntu 的开发和生产环境可能受到更新延迟的影响；结合 Copy Fail 漏洞，Linux 安全形势严峻
- **行动建议**：确保有备用的包镜像源；检查 CI/CD 管道中对 Ubuntu 官方源的依赖
