### 2. GitHub 关键 RCE 漏洞 CVE-2026-3854：一次 git push 即可入侵后端

- **来源**：[Wiz 安全博客](https://www.wiz.io/blog/github-rce-vulnerability-cve-2026-3854) / [GitHub 官方博客](https://github.blog/security/securing-the-git-push-pipeline-responding-to-a-critical-remote-code-execution-vulnerability/) / [The Hacker News](https://thehackernews.com/2026/04/researchers-discover-critical-github.html) / [HN 255 分 63 评论](https://news.ycombinator.com/item?id=47936479)
- **标签**：`GitHub` `安全` `CVE` `RCE` `git push` `Wiz`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：Wiz 安全研究人员通过 AI 辅助逆向工程发现 GitHub.com 和 GitHub Enterprise Server 的关键 RCE 漏洞（CVE-2026-3854，CVSS 8.7）。该漏洞为命令注入类型，任何拥有仓库 push 权限的认证用户仅需一次标准 `git push` 命令即可在 GitHub 后端服务器上执行任意命令，潜在影响数百万公共和私有仓库。GitHub 在不到两小时内完成验证、修复和调查，确认未被利用。
- **影响**：这是 GitHub 历史上最严重的安全漏洞之一：(1) 攻击面极广——任何认证用户都可能利用，且使用标准 git 客户端即可；(2) 跨租户风险——理论上可访问其他用户和组织的仓库；(3) GitHub 的快速响应（<2 小时修复）值得肯定，但漏洞本身的存在引发了对 GitHub 基础设施安全性的质疑。结合 Ghostty 离开 GitHub 的决定，这一天对 GitHub 来说是"完美风暴"。
- **行动建议**：(1) 如果你使用 GitHub Enterprise Server，立即检查是否已更新到修复版本；(2) 审查你的仓库 push 权限设置，确保最小权限原则；(3) 关注 GitHub 后续的安全改进措施。

### 3. GitHub Copilot 6 月 1 日起转向按量计费：AI 工具定价模式转折点

- **来源**：[Ars Technica](https://arstechnica.com/ai/2026/04/github-will-start-charging-copilot-users-based-on-their-actual-ai-usage/) / [InfoWorld](https://www.infoworld.com/article/4164236/github-shifts-copilot-to-usage-based-billing-signaling-new-cost-model-for-enterprise-ai-tools.html) / [Gamasutra](http://gamasutra.com/production/github-copilot-is-pivoting-to-usage-based-billing-because-it-s-currently-unsustainable)
- **标签**：`GitHub` `Copilot` `计费` `AI工具` `定价`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：GitHub 宣布从 6 月 1 日起，Copilot 将从固定订阅制转向基于 token 使用量的按量计费模式。月费不变（Pro $10、Pro+ $39、Business $19/用户、Enterprise $39/用户），但引入"GitHub AI Credits"系统，1 credit = $0.01。代码自动补全保持无限使用，但代码审查、Agent 模式等高级功能将消耗 credits。GitHub 表示无法继续承担"最重度 AI 用户的不断攀升的推理成本"。
- **影响**：这标志着 AI 开发工具从"包月制"向"按量制"的行业性转变：(1) 对重度用户（尤其是使用 Agent 模式的开发者）成本可能显著增加；(2) 这与 OpenAI 营收未达标的消息形成呼应——AI 工具的"免费午餐"时代正在结束；(3) 对企业 IT 预算管理提出新挑战——AI 工具成本从固定变为可变。Gamasutra 直言这是因为当前模式"不可持续"。
- **行动建议**：(1) 在 6 月 1 日前评估你的 Copilot 使用模式，了解哪些功能消耗 credits；(2) 考虑是否需要调整 AI 编码工具策略（如混合使用 Copilot + 本地模型）。
