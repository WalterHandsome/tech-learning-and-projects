### 1. AWS Interconnect 多云互联正式 GA：AWS-Google Cloud 私有连接，Azure 年内跟进

- **来源**：[AWS 官方博客](https://aws.amazon.com/blogs/aws/aws-weekly-roundup-claude-opus-4-7-in-amazon-bedrock-aws-interconnect-ga-and-more-april-20-2026/) / [TechTarget](https://www.techtarget.com/searchCloudComputing/news/366641818/Multi-cloud-networking-gains-momentum-with-AWS-and-Oracle) / [Oracle 公告](https://www.oracle.com/news/announcement/oracle-and-aws-collaborate-to-expand-multicloud-networking-2026-04-16/)
- **标签**：`AWS` `多云` `基础设施`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：AWS Interconnect - multicloud 于 4 月 14 日正式 GA，提供 AWS 与其他云服务商之间的托管私有连接，无需 VPN 或 colocation。首批支持 Google Cloud，Microsoft Azure 将在 2026 年晚些时候跟进。Oracle 同日宣布将 Oracle Interconnect 与 AWS Interconnect 对接，支持 OCI-AWS 之间的全栈和分栈多云部署。这是云计算行业"多云互联"从概念走向现实的标志性事件——三大云厂商首次通过开放互操作规范实现原生私有连接。
- **影响**：这彻底改变了多云架构的网络拓扑。此前，跨云连接需要通过 VPN、专线或第三方 colocation，延迟高、成本大、运维复杂。AWS Interconnect 提供了原生的、托管的、高速私有连接，大幅降低了多云部署的门槛。对于运行 AWS + GCP 混合架构的团队（例如在 AWS 上运行应用、在 GCP 上使用 BigQuery 或 TPU），这是一个立即可用的基础设施升级。
- **行动建议**：(1) 如果你的架构跨 AWS 和 GCP，评估 AWS Interconnect 替代现有 VPN/专线方案；(2) 关注 Azure 支持的时间线；(3) 评估 Oracle Interconnect 对接对混合云策略的影响。

### 3. Amazon S3 Files GA：S3 桶可直接挂载为 NFS 文件系统，~1ms 延迟

- **来源**：[AWS 官方公告](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-s3-files/) / [The New Stack](https://thenewstack.io/aws-s3-files-filesystem/) / [computingforgeeks](https://computingforgeeks.com/amazon-s3-files-guide-setup-performance-pricing/)
- **标签**：`AWS` `S3` `存储`
- **评分**：时效 4 / 一手 5 / 相关 5 / 实用 5 = 总分 19
- **摘要**：Amazon S3 Files 于 4 月 7 日正式 GA，这是首个将云对象存储直接暴露为高性能文件系统的服务。任何 S3 通用桶都可以通过 NFS v4.2 挂载为共享文件系统，支持 EC2、Lambda、EKS、ECS 等所有 AWS 计算资源。数据始终留在 S3 中，无需迁移或代码修改。延迟约 1ms，支持标准 POSIX 文件操作（open/read/write/rename）。这消除了对象存储和文件系统之间的传统权衡——你可以同时享受 S3 的持久性/成本优势和文件系统的交互性能。
- **影响**：这是 AWS 存储架构的范式转变。此前，需要文件系统语义的应用必须使用 EFS 或 FSx，数据与 S3 分离。S3 Files 让遗留应用无需修改即可直接访问 S3 数据，大幅简化了数据管道和 AI/ML 工作流。对于 Agent 系统来说，S3 Files 提供了一个理想的持久化工作空间——Agent 可以像操作本地文件一样操作 S3 数据。
- **行动建议**：(1) 评估 S3 Files 替代 EFS 的可行性（成本通常更低）；(2) 对于 AI/ML 训练管道，测试 S3 Files 的读写性能；(3) 关注与 OpenAI Agents SDK Manifest 抽象层的集成可能性。
