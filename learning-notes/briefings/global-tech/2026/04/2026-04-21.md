### 1. Vercel 遭入侵：Context.ai OAuth 链式攻击暴露客户凭证

- **来源**：[Vercel 官方公告](https://vercel.com/kb/bulletin/vercel-april-2026-security-incident) / [The Register](https://www.theregister.com/2026/04/20/vercel_context_ai_security_incident/) / [The Verge](https://www.theverge.com/tech/914723/vercel-hacked) / [CyberNews](https://cybernews.com/security/vercel-hacked-after-oauth-compromise/)
- **标签**：`安全事件` `供应链` `Vercel`
- **评分**：时效 5 / 一手 5 / 相关 5 / 实用 5 = 总分 20
- **摘要**：Vercel 于 4 月 19 日披露了一起安全事件：攻击者通过入侵第三方 AI 工具 Context.ai，利用 OAuth token 接管了一名 Vercel 员工的 Google Workspace 账户，进而访问了部分 Vercel 内部环境和未标记为"sensitive"的环境变量。黑客组织 ShinyHunters 正在出售窃取的数据，声称这可能成为"有史以来最大的供应链攻击"。Vercel 已联系受影响客户并建议立即轮换凭证。加密货币项目开发者尤其紧张，纷纷锁定 API 密钥。
- **影响**：这是 2026 年最重要的云平台安全事件之一。Vercel 是 Next.js 的维护者和主要托管平台，大量 Web 应用依赖其部署。攻击链的起点是一个 AI 工具的 OAuth 权限过宽（"Allow all"），这再次证明 AI 工具集成正在成为企业安全的新薄弱环节。对开发者的直接影响：如果你在 Vercel 上部署项目，立即检查环境变量是否标记为"sensitive"，轮换所有凭证。
- **行动建议**：(1) 审计 Vercel 项目中的环境变量，将敏感信息标记为"sensitive"；(2) 轮换所有 API 密钥和数据库凭证；(3) 检查 Google Workspace 审计日志中的异常登录；(4) 审查所有第三方 OAuth 集成的权限范围。

### 2. Meta 发布 Muse Spark：首个闭源模型，开源战略出现分水岭

- **来源**：[humai.blog](https://www.humai.blog/meta-released-muse-spark-its-first-closed-model-here-is-what-it-gave-up-to-do-it/) / [thenextgentechinsider](https://www.thenextgentechinsider.com/pulse/meta-stands-by-open-source-amid-shift-to-proprietary-models) / [implicator.ai](https://www.implicator.ai/meta-plans-to-open-source-new-ai-models-but-will-keep-its-most-powerful-closed/)
- **标签**：`Meta` `大模型` `开源`
- **评分**：时效 4 / 一手 4 / 相关 5 / 实用 4 = 总分 17
- **摘要**：Meta 于 4 月 8 日发布 Muse Spark，这是其历史上首个闭源大模型。Muse Spark 具备高级多 Agent 推理和多模态能力，仅通过私有 API 访问，不公开权重、架构或训练方法。与此同时，Llama 4 仍保持开源。Meta 将此定位为"混合策略"——开源模型服务开发者社区，闭源模型服务企业高端需求。背景是 DeepSeek 利用 Llama 代码构建竞品，以及 Llama 4 发布后开发者反响不佳。
- **影响**：这标志着 Meta 开源 AI 战略的重大转折。从 Llama 1 到 Llama 4，Meta 一直是开源 AI 的最大推动者。Muse Spark 的发布意味着 Meta 认为纯开源路线在商业上不可持续——最强模型需要闭源来保护竞争优势。对开发者来说，Llama 系列仍然可用，但 Meta 最强的能力将不再免费获取。

### 3. TeamPCP 供应链攻击：Trivy、LiteLLM、Axios 等开发工具被连环入侵

- **来源**：[Exposure Security](https://exposuresecurity.com/briefings/executive-briefing-teampcp-supply-chain-attack-v3-0-web-version.html) / [CISA/KEV](https://www.thenextgentechinsider.com/pulse/cisa-adds-critical-trivy-supply-chain-vulnerability-to-known-exploited-catalog) / [parseable.com](https://www.parseable.com/blog/supply-chain-audit-monitoring-otel-parseable)
- **标签**：`供应链安全` `CVE` `开发工具`
- **评分**：时效 4 / 一手 5 / 相关 5 / 实用 5 = 总分 19
- **摘要**：2026 年 3 月，威胁组织 TeamPCP 发动了一场级联式供应链攻击，先后入侵了四个广泛使用的开发工具：Trivy（安全扫描器）、Checkmarx KICS、LiteLLM（LLM 代理）和 Telnyx。攻击者通过被入侵的 Docker Hub 和 GitHub Actions 注入凭证窃取恶意软件，估计影响超过 50 万台机器。CISA 已将 Trivy 漏洞（CVE-2026-33634）加入 KEV 目录，联邦机构须在 4 月 9 日前修补。3 月 31 日，Axios（每周 1 亿+ npm 下载）也被入侵，攻击者在官方 npm 包中注入了远程访问木马。
- **影响**：这是 2026 年迄今最大规模的开发工具供应链攻击。讽刺的是，Trivy 本身就是一个安全扫描工具——"防御工具变成攻击武器"。对所有使用 npm/Docker 生态的开发者来说，这意味着即使是知名的安全工具也不能盲目信任。
- **行动建议**：(1) 检查项目中是否使用了受影响的 Trivy、LiteLLM、Axios 版本；(2) 审计 GitHub Actions 工作流中的第三方 Action；(3) 锁定依赖版本，启用 npm/Docker 签名验证；(4) 轮换可能暴露的凭证。
